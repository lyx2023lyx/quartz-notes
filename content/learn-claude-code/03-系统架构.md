## 📦 核心模块详解

### 1. 对话管理器 (ConversationManager)

> [!info] 职责
> 管理对话生命周期，协调各模块工作。

```mermaid
classDiagram
    class ConversationManager {
        +messages: List
        +tools: ToolRegistry
        +llm: LLMInterface
        +start()
        +process_input(user_input)
        +get_history()
        +clear_context()
    }
    
    class Message {
        +role: str
        +content: str
        +timestamp: datetime
    }
    
    ConversationManager "1" *-- "*" Message
```

**核心功能：**
- 维护对话历史
- 管理消息队列
- 处理上下文截断

### 2. 决策引擎 (DecisionEngine)

```mermaid
flowchart TD
    Input[用户输入] --> Parse[解析意图]
    Parse --> Think[LLM思考]
    Think --> Decision{决策类型}
    
    Decision -->|直接回答| Direct[生成回复]
    Decision -->|需要工具| ToolCall[调用工具]
    Decision -->|需要澄清| Clarify[请求澄清]
    
    ToolCall --> Execute[执行工具]
    Execute --> Observe[观察结果]
    Observe --> ThinkAgain[再次思考]
    ThinkAgain --> Decision
    
    Direct --> Output[输出给用户]
    Clarify --> Output
    
    style Decision fill:#ffcc80
```

## 🔄 主流程设计

### Agent主循环

```mermaid
sequenceDiagram
    participant User as 用户
    participant Agent as Agent主循环
    participant LLM as LLM接口
    participant Tools as 工具集
    
    loop 对话持续
        User->>Agent: 输入消息
        Agent->>Agent: 添加到历史
        
        loop ReAct循环
            Agent->>LLM: 发送prompt+历史
            LLM-->>Agent: 返回决策
            
            alt 需要工具调用
                Agent->>Agent: 解析工具请求
                Agent->>Tools: 执行工具
                Tools-->>Agent: 返回结果
                Agent->>Agent: 添加到历史
            else 直接回答
                Agent->>Agent: 准备最终回复
            end
        end
        
        Agent->>User: 显示回复
    end
```

### 详细流程图

```mermaid
stateDiagram-v2
    [*] --> Idle: 启动Agent
    Idle --> Processing: 接收用户输入
    
    Processing --> Thinking: 构建Prompt
    Thinking --> LLMCalling: 调用LLM
    LLMCalling --> Decision: 获取响应
    
    Decision --> ToolExecution: 检测到工具调用
    Decision --> Responding: 直接回复
    
    ToolExecution --> Validation: 验证参数
    Validation --> Executing: 参数合法
    Validation --> Error: 参数非法
    
    Executing --> Observation: 执行完成
    Observation --> Thinking: 继续思考
    
    Error --> Thinking: 报告错误
    
    Responding --> Idle: 显示回复
    
    Idle --> [*]: 退出命令
    
    note right of Thinking
        ReAct模式的核心：
        思考 -> 行动 -> 观察
    end note
```

## 📝 数据结构定义

### 消息格式

```python
{
    "role": "user" | "assistant" | "system" | "tool",
    "content": "消息内容",
    "tool_calls": [  # assistant消息可能有
        {
            "id": "call_123",
            "type": "function",
            "function": {
                "name": "read_file",
                "arguments": '{"path": "main.py"}'
            }
        }
    ],
    "tool_call_id": "call_123",  # tool消息需要
    "name": "read_file"  # tool消息需要
```

## 📊 性能考虑

### 优化策略

| 优化点 | 策略 | 效果 |
|--------|------|------|
| Token使用 | 上下文压缩、摘要 | 降低成本 |
| 响应速度 | 流式输出、并行工具 | 提升体验 |
| 工具调用 | 结果缓存 | 减少重复执行 |
| 错误恢复 | 重试机制 | 提高稳定性 |

> [!tip] 架构设计原则
> 1. **模块化**：每个职责独立，便于测试和维护
> 2. **可扩展**：新工具可轻松添加
> 3. **可配置**：关键参数可外部配置
> 4. **安全性**：多层防护，默认安全
