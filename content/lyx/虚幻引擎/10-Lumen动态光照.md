# 10 Lumen 动态全局光照

> 🎯 **本章目标**：理解Lumen原理，掌握Lumen光照设置和优化

## 一、Lumen 是什么？

### 1.1 通俗解释

```
【传统光照的问题】

传统光照：
• 光照烘焙 = 预计算，静态的
• 动态光照 = 实时计算，但效果差
• 间接光 = 很难计算，通常用假效果

┌─────────────────────────────────────────────┐
│  传统动态光照效果：                          │
│                                             │
│       💡 光源                                │
│        ↓ 直接光照                           │
│    ┌───────┐                                │
│    │  亮   │  ← 直接照亮                    │
│    │  暗   │  ← 没有间接光，全黑            │
│    └───────┘                                │
│                                             │
│  阴影区域死黑，不真实                        │
└─────────────────────────────────────────────┘

【Lumen 解决方案】

Lumen = 完全动态的全局光照系统

• 光线实时反弹（间接光照）
• 光源可以任意移动
• 物体移动时光照自动更新
• 接近烘焙质量的动态效果

┌─────────────────────────────────────────────┐
│  Lumen 效果：                                │
│                                             │
│       💡 光源                                │
│        ↓ 直接光照                           │
│    ┌───────┐                                │
│    │  亮   │  ← 直接照亮                    │
│    │ 反射光│  ← 墙壁反射的光！              │
│    └───────┘                                │
│                                             │
│  光线在场景里真实反弹，阴影有细节            │
└─────────────────────────────────────────────┘
```

### 1.2 Lumen 的核心特性

```
┌─────────────────────────────────────────────────────────┐
│                   Lumen 三大特性                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1️⃣ 动态全局光照 (Global Illumination)                 │
│     • 光线多次反弹                                      │
│     • 真实的间接光照                                    │
│     • 移动的物体也正确受光                              │
│                                                         │
│  2️⃣ 动态反射 (Reflections)                             │
│     • 真实的镜面反射                                    │
│     • 粗糙表面的漫反射                                  │
│     • 无需反射球/反射平面                               │
│                                                         │
│  3️⃣ 实时光线追踪效果                                   │
│     • 软件光线追踪（无需RTX显卡）                       │
│     • 硬件光线追踪加速（RTX显卡更好）                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.3 Lumen vs 传统光照

```
┌─────────────────────────────────────────────────────────┐
│                Lumen vs 传统光照                        │
├────────────────┬────────────────┬───────────────────────┤
│ 特性            │ 传统烘焙        │ Lumen 动态           │
├────────────────┼────────────────┼───────────────────────┤
│ 光源移动        │ ❌ 不能动       │ ✅ 任意移动          │
│ 物体移动        │ ❌ 光照不变     │ ✅ 实时更新          │
│ 间接光          │ ✅ 烘焙好       │ ✅ 实时计算          │
│ 反射            │ ❌ 静态反射球   │ ✅ 动态真实反射      │
│ 构建时间        │ ❌ 烘焙耗时     │ ✅ 无需烘焙          │
│ 运行时性能      │ ✅ 性能最好     │ ⚠️ 较高性能要求      │
│ 质量            │ ✅ 最高         │ ✅ 接近烘焙质量      │
└────────────────┴────────────────┴───────────────────────┘

💡 结论：
• 开发迭代：用 Lumen（快速预览）
• 发布平台性能强：用 Lumen
• 发布平台性能弱：烘焙 + Lumen反射
```

## 二、启用 Lumen

### 2.1 项目设置

```
【启用 Lumen】

步骤：
编辑 → 项目设置 → 渲染

┌─────────────────────────────────────────────┐
│ Rendering                                   │
├─────────────────────────────────────────────┤
│ Dynamic Global Illumination Method           │
│   ○ None                                    │
│   ● Lumen ← 选这个                          │
│   ○ Screen Space (SSGI)                     │
│   ○ Ray Tracing (Deprecated)                │
├─────────────────────────────────────────────┤
│ Reflection Method                            │
│   ○ None                                    │
│   ● Lumen ← 选这个                          │
│   ○ Screen Space                            │
│   ○ Ray Tracing                             │
├─────────────────────────────────────────────┤
│ Software Ray Tracing Mode                    │
│   ● Global SDF ← 推荐                       │
│   ○ Heightfield                               │
└─────────────────────────────────────────────┘

其他重要设置：
☑ Generate Mesh Distance Fields
  （必须开启，Lumen依赖距离场）
```

### 2.2 关卡设置

```
【后处理体积设置】

Lumen 设置通常在后处理体积里：

放置Actor → 视觉效果 → 后处理体积

细节面板 → 全局光照：
┌─────────────────────────────────────────────┐
│ Global Illumination                         │
├─────────────────────────────────────────────┤
│ Method: Lumen                               │
│ Final Gather Quality: 1 ← 质量（1-4）       │
│ Final Gather Distance: 20000                │
├─────────────────────────────────────────────┤
│ Reflections                                 │
├─────────────────────────────────────────────┤
│ Method: Lumen                               │
│ Quality: 1 ← 质量（0-4）                    │
│ Ray Tracing Reflections: Disabled           │
└─────────────────────────────────────────────┘

💡 后处理体积默认无限范围（Infinite Extent）
   如果没有后处理体积，项目设置生效
```

### 2.3 验证 Lumen 工作

```
【检查 Lumen 是否生效】

方法1：看控制台命令
   按 ~ 键打开控制台
   输入：stat gpu
   查看是否有 Lumen 相关开销

方法2：看画面效果
   • 移动光源，观察反射是否实时变化
   • 看阴影区域是否有间接光照亮
   • 看镜面反射是否清晰

方法3：强制关闭对比
   控制台：r.DynamicGlobalIlluminationMethod 0
   （0=关闭，1=Lumen）
```

## 三、Lumen 核心概念

### 3.1 全局光照原理

```
【光线如何反弹】

┌─────────────────────────────────────────────┐
│                                             │
│  光源 💡                                    │
│    │                                        │
│    │ 1️⃣ 直接光（Primary）                   │
│    ▼                                        │
│  ┌────┐  → 直接照亮                        │
│  │红墙│                                    │
│  └─┬──┘                                    │
│    │                                        │
│    │ 2️⃣ 第一次反弹（Secondary）             │
│    │    红墙反射红光                       │
│    ▼                                        │
│  ┌────┐  → 白色物体变红                    │
│  │白球│  （颜色渗透）                      │
│  └──┬─┘                                    │
│     │                                       │
│     │ 3️⃣ 第二次反弹                        │
│     ▼                                       │
│   ...                                       │
│                                             │
│ Lumen 计算多次反弹，结果更真实              │
└─────────────────────────────────────────────┘
```

### 3.2 距离场（Distance Fields）

```
【什么是距离场？】

距离场 = 每个点到最近物体的距离

┌─────────────────────────────────────────────┐
│  场景：                                      │
│  ┌───────────┐                              │
│  │    ⭕     │  ← 球体                      │
│  │  (球心)   │                              │
│  └───────────┘                              │
│                                             │
│  距离场表示：                                │
│  • 球心：距离最近表面 = 半径（最大正值）    │
│  • 球内：负值（在物体内）                   │
│  • 球外：正值，越远越大                     │
│                                             │
│  ╭──────────────╮                          │
│  │   ⬜ ⬜ ⬜    │  ← 远处（大数值）        │
│  │   ⬜ ⭕ ⬜    │  ← 近处（小数值）        │
│  │   ⬜ ⬜ ⬜    │                          │
│  ╰──────────────╯                          │
│                                             │
│ Lumen 用距离场快速计算光线追踪              │
│ 不需要逐三角形计算，速度快！                │
└─────────────────────────────────────────────┘

【距离场设置】
项目设置 → 渲染 → 生成网格距离场 ☑

距离场质量：
控制台：r.DFShadowQuality 1-5
```

### 3.3 表面缓存（Surface Cache）

```
【Lumen 如何存储光照信息】

Lumen 使用 Surface Cache 存储光照：
• 在物体表面生成"光照贴图"
• 但这些贴图是动态更新的
• 不需要UV展开

┌─────────────────────────────────────────────┐
│                                             │
│   3D模型表面 → 自动生成 Surface Cache      │
│                                             │
│   ┌─────────┐                               │
│   │ ▓▓░░▓▓▓ │  ← 存储光照亮度/颜色         │
│   │ ▓░░░░▓▓ │                               │
│   │ ▓▓▓░░░▓ │  实时更新，无需烘焙          │
│   └─────────┘                               │
│                                             │
│ 查看 Surface Cache：                        │
│ 视口 → 显示 → Lumen → Surface Cache        │
│                                             │
└─────────────────────────────────────────────┘
```

### 4.1 全局设置

```

控制台命令或项目设置：

┌─────────────────────────────────────────────┐
│ Lumen 全局光照质量                          │

│                                             │

│                                             │
│ r.Lumen.Reflections.MaxReflectionBounces 1-8│
│   反射反弹次数                              │
│   镜子对面有镜子需要更多                    │
└─────────────────────────────────────────────┘
```

## 五、Lumen 优化

### 5.1 性能影响因素

```
【什么影响 Lumen 性能？】

1. 屏幕分辨率
   • 分辨率越高，计算量越大
   • 4K 比 1080p 慢 4 倍

2. 场景复杂度
   • 物体数量多 → Surface Cache 大
   • 复杂几何 → 距离场计算慢

3. 反射数量
   • 镜面反射表面多 → 更慢
   • 粗糙反射相对快

4. 质量设置
   • Final Gather Quality 每级性能差距大
   • 反射反弹次数

5. 视口覆盖
   • Lumen 主要计算屏幕可见区域
   • 大世界影响相对小
```

### 5.2 优化技巧

```
┌─────────────────────────────────────────────────────────┐
│                    Lumen 优化清单                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 分辨率缩放                                          │
│     r.ScreenPercentage 75 ← 以75%分辨率渲染Lumen        │
│     再放大，视觉损失小，性能提升大                      │
│                                                         │
│  2. 控制反射表面                                        │
│     • 非必要不做镜面反射                                │
│     • 降低 Roughness < 0.2 的表面数量                  │
│                                                         │
│  3. 简化距离场                                          │
│     • 复杂小物件关闭 "生成距离场"                       │
│     • 远距离物体用 LOD 简化                             │
│                                                         │
│  4. 分区质量                                            │
│     • 重点区域高质量                                    │
│     • 远处/边缘低质量                                   │
│                                                         │
│  5. 硬件光线追踪（如有RTX显卡）                         │
│     • r.Lumen.HardwareRayTracing 1                      │
│     • 质量更好，有时性能更好                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 5.3 距离场优化

```
【距离场设置】

单个物体设置：
选中物体 → 细节面板
┌─────────────────────────────────────────────┐
│ Rendering                                   │
├─────────────────────────────────────────────┤
│ ☐ Generate Distance Field                   │
│   ← 取消勾选 = 不参与 Lumen                │
│                                             │
│ Distance Field Self Shadow Bias: 0          │
│ Distance Field Resolution Scale: 1          │
│   ← 可以缩小减少开销                       │
└─────────────────────────────────────────────┘

【建议关闭距离场的物体】
• 很小的杂物（瓶子、石头）
• 叶子、草（植被）
• 透明物体
• 远距离物体
```

## 六、Lumen 常见问题

### 6.1 常见问题解决

```
❌ 画面有噪点/闪烁
    原因：Final Gather Quality 太低
    解决：提高质量或开启 Temporal Filtering
    命令：r.Lumen.FinalGatherQuality 2

❌ 反射模糊/不清楚
    原因：反射质量设置低
    解决：提高 r.Lumen.Reflections.MaxRoughnessToTrace
    或检查后处理体积反射质量

❌ 移动物体有残影
    原因：Surface Cache 更新延迟
    解决：正常现象，快速移动会有
    改善：r.Lumen.SceneLighting.MaxCardUpdatesPerFrame 128

❌ 远距离物体没有间接光
    原因：Lumen 主要计算屏幕内
    解决：配合 Sky Light 使用

❌ 黑色区域太多
    原因：光线被遮挡，无法反弹
    解决：
    • 增加光源
    • 提高 Final Gather Distance
    • 检查 Sky Light 设置
```

### 6.2 与其他系统配合

```
【Lumen + 烘焙光照】

可以混合使用：
• 静态物体用烘焙光照（性能最优）
• 动态物体用 Lumen
• 反射用 Lumen

设置：
光源 Mobility = Static
      ↓
烘焙直接光
      ↓
Lumen 提供间接光 + 反射

【Lumen + 后期处理】

后处理体积设置：
• Exposure 曝光控制
• Color Grading 调色
• Bloom 泛光
• Lumen 质量设置

建议顺序：
Lumen → 色调映射 → 曝光 → 调色
```
