# 11 Nanite 虚拟几何体

> 🎯 **本章目标**：理解Nanite原理，掌握Nanite使用方法和注意事项

---

## 一、Nanite 是什么？

### 1.1 通俗解释

```
【传统模型的问题】

做高精度模型面临的困境：

┌─────────────────────────────────────────────┐
│  高精度模型：                                │
│  ┌─────────────┐                            │
│  │  ⬡⬡⬡⬡⬡⬡⬡  │  1000万面                │
│  │  ⬡⬡⬡⬡⬡⬡⬡  │                            │
│  │  ⬡⬡⬡⬡⬡⬡⬡  │  效果极好，但是...        │
│  └─────────────┘                            │
│                                             │
│  问题：                                      │
│  ❌ 显卡跑不动（帧数极低）                   │
│  ❌ 内存放不下（显存爆炸）                   │
│  ❌ 需要手动做LOD（费时费力）                │
│                                             │
└─────────────────────────────────────────────┘

【Nanite 解决方案】

Nanite = 虚拟几何体 = "智能自动LOD"

特点：
• 直接导入超高精度模型（数亿面）
• 自动根据距离调整细节
• 没有LOD切换的"跳变"
• 性能几乎不受面数影响

┌─────────────────────────────────────────────┐
│  Nanite 效果：                               │
│                                             │
│  远处：            近处：                    │
│  ┌────┐           ┌─────────────┐          │
│  │▓▓▓▓│           │⬡⬡⬡⬡⬡⬡⬡⬡⬡│          │
│  │▓▓▓▓│           │⬡⬡⬡⬡⬡⬡⬡⬡⬡│          │
│  └────┘           │⬡⬡⬡⬡⬡⬡⬡⬡⬡│          │
│   简化            └─────────────┘          │
│                   完整细节                  │
│                                             │
│  无缝过渡，没有"突跳"                       │
│  性能稳定，不管模型多复杂                   │
└─────────────────────────────────────────────┘
```

### 1.2 Nanite 核心特性

```
┌─────────────────────────────────────────────────────────┐
│                   Nanite 核心特性                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1️⃣ 虚拟几何体（Virtual Geometry）                     │
│     • 将模型分割成微多边形（Micro-Polygons）            │
│     • 每个像素最多一个三角形                            │
│     • 根据屏幕像素动态选择细节                          │
│                                                         │
│  2️⃣ 自动LOD                                            │
│     • 无需手动制作LOD链                                 │
│     • 无限层级细节过渡                                  │
│     • 没有LOD切换瑕疵                                   │
│                                                         │
│  3️⃣ 智能剔除                                           │
│     • 看不见的像素不渲染                                │
│     • 自动遮挡剔除                                      │
│     • 大幅减少绘制调用（Draw Call）                     │
│                                                         │
│  4️⃣ 高压缩存储                                         │
│     • 比原始模型小很多                                  │
│     • 磁盘和内存都节省                                  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.3 Nanite vs 传统 LOD

```
┌─────────────────────────────────────────────────────────┐
│               Nanite vs 传统 LOD                        │
├────────────────┬────────────────┬───────────────────────┤
│ 特性            │ 传统 LOD        │ Nanite               │
├────────────────┼────────────────┼───────────────────────┤
│ 准备工作        │ 手动做LOD链     │ 直接导入高模         │
│ 切换效果        │ 明显"跳变"      │ 无缝平滑             │
│ 面数限制        │ 每级有固定面数  │ 无限制               │
│ 内存使用        │ 同时存多份模型  │ 一份智能数据         │
│ 磁盘空间        │ 大（多份LOD）   │ 小（高度压缩）       │
│ 支持功能        │ 全部支持        │ 部分不支持           │
│ 性能            │ 取决于LOD设置   │ 稳定流畅             │
└────────────────┴────────────────┴───────────────────────┘

💡 结论：
• 能开Nanite就开Nanite
• 注意不支持的特性
• 适合：建筑、石头、静态场景
```

## 二、启用 Nanite

### 2.1 导入时启用

```
【FBX 导入选项】

导入模型时：

┌─────────────────────────────────────────────┐
│ FBX 导入选项                                 │
├─────────────────────────────────────────────┤
│ 【几何体】                                   │
│   ☑ 构建 Nanite                             │
│      ← 勾选这个！                           │
│                                             │
│   Nanite 设置：                              │
│   • Position Precision: Auto                │
│     （顶点位置精度，自动即可）              │
│   • Normal Precision: Auto                  │
│     （法线精度）                            │
│   • Triangle Threshold: 2000                │
│     （小于此面数不启用Nanite）              │
│     ← 可以改小，让更多模型用Nanite          │
└─────────────────────────────────────────────┘

💡 建议：Triangle Threshold 设为 100 或更小
      让更多模型受益于 Nanite
```

### 2.2 导入后启用

```
【已有模型启用 Nanite】

方法1：静态网格体编辑器
1. 双击打开静态网格体
2. 细节面板 → Nanite 设置
3. ☑ 启用 Nanite 支持

方法2：批量启用
1. 内容浏览器多选模型
2. 右键 → Nanite → 启用

方法3：构建 Nanite
1. 选中模型
2. 右键 → Nanite → 重新构建

【Nanite 设置面板】
┌─────────────────────────────────────────────┐
│ Nanite Settings                              │
├─────────────────────────────────────────────┤
│ ☑ Enable Nanite Support                     │
│   ← 开关 Nanite                             │
│                                             │
│ Position Precision                           │
│   • 越低精度越高，文件越大                  │
│   • 默认 Auto 即可                          │
│                                             │
│ Normal Precision                             │
│   • 法线精度                                │
│   • 影响光照细节                            │
│                                             │
│ Triangle Percent                             │
│   • 保留的三角形百分比                      │
│   • 100% = 全部保留                         │
│   • 可以减小来降低质量                      │
└─────────────────────────────────────────────┘
```

### 2.3 验证 Nanite 工作

```
【检查 Nanite 是否生效】

方法1：看细节面板
   选中模型 → 看是否勾选了 Enable Nanite

方法2：可视化显示
   视口 → 显示 → Nanite → 三角形计数
   
   显示效果：
   • 绿色 = Nanite 模型
   • 蓝色 = 传统模型
   
方法3：统计数据
   控制台：stat nanite
   查看 Nanite 统计信息

方法4：观察效果
   • 靠近模型，看细节是否清晰
   • 远离模型，看是否自动简化
   • 过渡是否平滑无跳变
```

## 三、Nanite 工作原理

### 3.1 微多边形网格

```
【什么是微多边形？】

传统渲染：
• 模型三角形投影到屏幕
• 一个三角形可能只占几个像素
• 顶点处理浪费

Nanite 渲染：
• 确保每个像素最多一个三角形
• 三角形大小均匀
• 没有小于像素的三角形

┌─────────────────────────────────────────────┐
│                                             │
│  传统模型（问题）：                          │
│  ┌─────────────────────────┐               │
│  │ ⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡ │               │
│  │ ⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡⬡ │               │
│  │ ░░░░░░░░░░░░░░░░░░░░░░░ │               │
│  │ ░░小三角形浪费░░░░░░░░░ │               │
│  └─────────────────────────┘               │
│                                             │
│  Nanite（解决）：                            │
│  ┌─────────────────────────┐               │
│  │ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │               │
│  │ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │               │
│  │ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │               │
│  │ ▓▓每个像素一个三角形▓▓▓ │               │
│  └─────────────────────────┘               │
│                                             │
│  像素级别细节，无浪费                        │
└─────────────────────────────────────────────┘
```

### 3.2 数据流与剔除

```
【Nanite 数据流】

1. 数据存储
   • 模型分成小簇（Cluster）
   • 每簇存储多个细节级别
   • 高度压缩存储

2. 渲染流程
┌─────────────────────────────────────────────┐
│                                             │
│  ① 可见性测试                               │
│     判断哪些簇在视口内                      │
│              ↓                              │
│  ② 细节选择                                 │
│     根据距离选择合适LOD                     │
│     确保每个像素一个三角形                  │
│              ↓                              │
│  ③ 数据流送                                 │
│     只加载需要的簇到GPU                     │
│              ↓                              │
│  ④ 渲染                                     │
│     高效绘制                                │
│                                             │
└─────────────────────────────────────────────┘

3. 智能剔除
   • 视锥剔除：看不见的不处理
   • 遮挡剔除：被挡住的跳过
   • 细节剔除：远距离用简化版
```

```

## 五、Nanite 限制与解决

### 5.1 功能限制清单

```
┌─────────────────────────────────────────────────────────┐
│                  Nanite 不支持的功能                     │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ❌ 透明材质（Translucent）                             │
│     • 玻璃、水、半透物体                                │
│     • 树叶（通常用Masked）                              │
│                                                         │
│  ❌ 蒙版材质（Masked）部分功能                          │
│     • 可以渲染，但遮挡剔除可能不准                      │
│                                                         │
│  ❌ 顶点动画                                            │
│     • 骨骼动画                                          │
│     • 顶点偏移（WPO在Nanite上有限制）                   │
│                                                         │
│  ❌ 特定材质特性                                        │
│     • 曲面细分（Tessellation）                          │
│     • 像素深度偏移（Pixel Depth Offset）                │
│                                                         │
│  ❌ 特定光照特性                                        │
│     • 顶点光照                                          │
│     • 光照函数                                          │
│                                                         │
│  ⚠️ 部分支持但需谨慎                                    │
│     • 顶点颜色（支持）                                  │
│     • 自定义深度/模板（支持）                           │
│     • 距离场（支持）                                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 5.2 解决方案

```
【问题1：需要做透明物体】

解决：关闭 Nanite
1. 静态网格体编辑器
2. 取消勾选 Enable Nanite Support
3. 确保有碰撞体

【问题2：树叶需要风动】

解决1：用传统 LOD
• 关闭 Nanite
• 手动做 LOD 链
• 材质用 SimpleGrassWind

解决2：用 WPO + Nanite
• 保持 Nanite 开启
• 材质使用 World Position Offset
• 注意：大范围 WPO 可能影响剔除

【问题3：需要顶点位移】

解决：
• 小范围位移：材质 WPO 可用
• 大范围变形：关闭 Nanite
• 动画：用骨骼网格体

【问题4：碰撞问题】

Nanite 不自动生成碰撞！
解决：
• 单独设置碰撞（Convex Decomposition）
• 或用简单碰撞体
```

## 六、Nanite 优化

### 6.1 性能优化

```
┌─────────────────────────────────────────────────────────┐
│                    Nanite 优化建议                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 合理设置 Triangle Threshold                         │
│     • 太小（100）：太多小物体用 Nanite，可能负收益     │
│     • 太大（5000）：只有超大模型用，错过优化机会        │
│     • 推荐：200-1000                                    │
│                                                         │
│  2. 控制 Position Precision                             │
│     • 默认 Auto 即可                                    │
│     • 高精度需求：减小数值                              │
│     • 远距离物体：增大数值节省空间                      │
│                                                         │
│  3. 结合传统渲染                                        │
│     • 远距离用 Nanite 简化                              │
│     • 中距离用 Nanite                                   │
│     • 近距离特殊物体关闭 Nanite                         │
│                                                         │
│  4. 遮挡剔除                                            │
│     • Nanite 自动处理遮挡                               │
│     • 合理设计场景，避免过度绘制                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

```

## 七、实战技巧

### 7.1 地编 Nanite 工作流

```
【推荐工作流】

1. 导入阶段
   • 导入高模
   • 开启 Nanite（Triangle Threshold 200）
   • 检查导入结果

2. 设置阶段
   • 设置碰撞（必须手动！）
   • 调整材质
   • 测试 Nanite 开关效果

3. 场景搭建
   • 放置 Nanite 模型
   • 混合使用传统模型
   • 检查性能

4. 优化阶段
   • 观察可视化和统计
   • 调整不适合的模型
   • 平衡质量和性能

【混合场景示例】
┌─────────────────────────────────────────────┐
│                                             │
│  场景组成：                                  │
│  • 建筑主体 = Nanite（高模扫描）            │
│  • 地面 = 地形系统                          │
│  • 树木 = 传统 LOD（风动画）                │
│  • 窗户玻璃 = 关闭 Nanite 的静态网格        │
│  • 草 = 草地系统（Grass Type）              │
│                                             │
│  各种技术结合，取长补短                      │
└─────────────────────────────────────────────┘
```

### 7.2 扫描模型工作流

```
【Quixel Megascans + Nanite】

步骤：
1. Bridge 导入 Megascans 资源
   • 自动开启 Nanite
   • 自动设置材质

2. 检查设置
   • 确认 Nanite 已启用
   • 检查碰撞是否生成

3. 场景使用
   • 直接拖拽到场景
   • 配合地形使用
   • 大量实例化也没问题

优势：
• Megascans 资源超高精度
• Nanite 保证性能
• 即拖即用
```
