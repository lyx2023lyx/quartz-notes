# 04 项目与关卡基础

> 🎯 **本章目标**：理解项目和关卡的关系，学会创建和管理关卡

---

## 一、项目 vs 关卡

### 1.1 通俗比喻

```
【项目 Project】= 一个完整的游戏/作品
就像一部电影：
├── 有自己的名字（项目名称）
├── 有自己的文件夹（项目目录）
├── 包含所有资源（模型、贴图、材质）
└── 包含一个或多个关卡

【关卡 Level】= 项目里的一个场景
就像电影里的一个镜头/场景：
├── 关卡1：新手村
├── 关卡2：森林地图  
└── 关卡3：城堡决战

关系：
📁 项目《我的游戏》
   ├── 📄 关卡1 - 村庄
   ├── 📄 关卡2 - 森林
   └── 📄 关卡3 - 城堡
```

### 1.2 实际例子

| 游戏 | 项目 | 关卡示例 |
|------|------|----------|
| 原神 | GenshinImpact | 蒙德城、璃月港、稻妻 |
| 王者荣耀 | HonorOfKings | 王者峡谷、大乱斗地图 |
| 黑神话 | BlackMyth | 黑风山、黄风岭 |

## 二、项目结构详解

### 2.1 项目文件夹结构

```
我的项目/
├── 📁 Config/              ← 配置文件（不用管）
├── 📁 Content/             ← ⭐ 你的所有资源在这里！
│   ├── 📁 Maps/           ← 关卡文件存放地
│   │   └── 🗺️ Level_1.umap
│   ├── 📁 Materials/      ← 材质
│   ├── 📁 Meshes/         ← 模型
│   ├── 📁 Textures/       ← 贴图
│   └── 📁 Blueprints/     ← 蓝图
├── 📁 Intermediate/        ← 临时文件（不用管）
├── 📁 Saved/              ← 自动保存、日志
│   ├── 📁 Autosaves/      ← 自动备份
│   ├── 📁 Logs/           ← 日志
│   └── 📁 Screenshots/    ← 截图
├── 📁 Source/             ← C++代码（蓝图项目可空）
└── 📄 我的项目.uproject   ← 项目文件，双击打开
```

💡 **重点**：地编主要在 `Content/` 目录里工作

### 2.2 Content 文件夹组织建议

```
Content/
├── 📁 _Common/            ← 公共资源（前缀_放上面）
│   ├── 📁 Materials/      ← 基础材质
│   ├── 📁 Meshes/         ← 通用模型
│   └── 📁 Textures/       ← 通用贴图
├── 📁 Maps/               ← 所有关卡
│   ├── 🗺️ L_MainMenu.umap
│   ├── 🗺️ L_Level_01.umap
│   └── 🗺️ L_Test.umap
├── 📁 Environment/        ← 环境相关
│   ├── 📁 Landscapes/     ← 地形
│   ├── 📁 Foliage/        ← 植被
│   └── 📁 Lighting/       ← 灯光相关
└── 📁 Props/              ← 道具
    ├── 📁 Architecture/   ← 建筑
    └── 📁 Decorations/    ← 装饰物
```

💡 **命名规范**：
- 关卡前缀 `L_` → L_Level_01
- 材质前缀 `M_` → M_Wood
- 静态网格前缀 `SM_` → SM_Chair
- 贴图前缀 `T_` → T_Wood_D

## 四、关卡编辑器详解

### 4.1 默认关卡包含什么？

新建"默认"关卡后，大纲里会有：

```
大纲视图
├── 📁 Lighting                    ← 光照文件夹
│   ├── ☀️ DirectionalLight        ← 主光源（太阳）
│   ├── 💡 SkyLight               ← 天空光（环境光）
│   └── 🌫️ SkyAtmosphere          ← 大气系统
│
├── 📁 LevelInstance               ← 关卡实例（暂时不用管）
├── 🎮 PlayerStart                ← 玩家出生点
│                                   （游戏从这里开始）
│
├── ⬜ Floor                      ← 默认地板
│   （一个平面，站在上面不会掉下去）
│
└── 📁 NavMeshBoundsVolume         ← 导航网格（AI用）
```

### 4.2 关键物体说明

| 物体 | 图标 | 作用 | 能否删除 |
|------|------|------|----------|
| **DirectionalLight** | ☀️ | 主光源/太阳 | ⚠️ 删了会全黑 |
| **SkyLight** | 💡 | 环境光照/天空色 | ⚠️ 删了环境变黑 |
| **SkyAtmosphere** | 🌫️ | 大气效果/天空盒 | 可删 |
| **PlayerStart** | 🎮 | 游戏开始位置 | 可删 |
| **Floor** | ⬜ | 默认地板 | 可删 |

### 4.3 关卡必备元素清单

一个可正常运行的关卡至少要有：

```
✅ 至少一个光源（DirectionalLight 或 PointLight）
✅ SkyLight（获得天空颜色）
✅ 玩家出生点 PlayerStart
✅ 地板或地形（不然会一直掉落）
```

## 五、关卡操作实战

### 5.1 添加基础物体

```
方法一：放置Actor面板
窗口 → 放置Actor
  ├── 基础 → 立方体/球体/圆柱
  ├── 光照 → 各种灯光
  └── 几何体 → 笔刷（临时阻挡）

方法二：内容浏览器拖拽
找到资源 → 拖到场景中

方法三：大纲右键
右键 → 放置Actor
```

### 5.2 场景搭建示例

```
【任务】搭建一个简单的房间

步骤：
1. 删除默认的 Floor（选中按 Delete）

2. 创建地板：
   放置Actor → 基础 → 立方体
   缩放 Z=0.1（变薄）
   缩放 X=20, Y=20（变大）

3. 创建墙壁：
   复制地板（Ctrl+W 或 Alt+拖拽）
   旋转90度立起来
   调整位置到边缘

4. 添加灯光：
   放置Actor → 光照 → 点光源
   拖到房间中央

5. 保存关卡：
   Ctrl+S → 命名 L_Room_Test
   保存到 Content/Maps/
```

### 5.3 关卡保存与版本

```
【手动保存】
Ctrl+S = 保存当前关卡
Ctrl+Shift+S = 另存为

【自动保存】
编辑 → 编辑器偏好设置 → 自动保存
  建议开启：每5-10分钟保存一次
  位置：项目/Saved/Autosaves/

【保存截图】
视口右上角相机图标 → 高分辨率截图
或 F9 键快速截图
```

## 六、多关卡项目管理

### 6.1 场景过渡

```
【场景切换概念】

玩家在游戏中会从一个关卡到另一个关卡：

主菜单关卡 (L_MainMenu)
       │ 点击"开始游戏"
       ▼
教学关卡 (L_Tutorial)
       │ 完成教学
       ▼  
第一关 (L_Level_01)
       │ 通关
       ▼
第二关 (L_Level_02)

【加载方式】
- 无缝加载：大世界分区，不读条
- 传统加载：黑屏读条
```

### 6.2 关卡流送（Level Streaming）

```
【概念】
把大场景拆成多个子关卡，按需加载

就像开放世界游戏：
┌──────────────────────────────────────┐
│           大世界地图                  │
│   ┌─────┬─────┬─────┐               │
│   │ 区域A │ 区域B │ 区域C │ ← 子关卡 │
│   ├─────┼─────┼─────┤               │
│   │ 区域D │ 玩家 │ 区域F │           │
│   ├─────┼─────┼─────┤               │
│   │ 区域G │ 区域H │ 区域I │           │
│   └─────┴─────┴─────┘               │
│                                       │
│   玩家走到哪，加载哪个区域            │
│   远离的区域卸载，节省内存            │
└──────────────────────────────────────┘

【地编应用】
- 超大场景必须拆分
- 团队协作时每人负责一个子关卡
```

### 6.3 子关卡操作

```
添加子关卡：
窗口 → 关卡
  → 点击【关卡】下拉菜单
  → 添加现有关卡 / 创建新关卡

子关卡显示方式：
- 蓝图（可视化编辑）
- 总是加载（常驻内存）
- 按需加载（根据触发器）
```

## 七、项目设置优化

### 7.1 编辑器偏好设置

```
编辑 → 编辑器偏好设置

【通用】
  自动保存：开启，间隔300秒
  自动保存目录：项目/Saved/Autosaves

【视口】
  使用飞行速度缩放：开启
  默认相机速度：4（可根据喜好调整）

【加载和保存】
  启动时加载上一个项目：建议开启
  自动重新导入：开启（外部修改文件自动更新）
```

### 7.2 项目设置

```
编辑 → 项目设置

【地图和模式】
  默认地图 → 编辑器开始地图
  默认地图 → 游戏默认地图
  
【渲染】
  动态全局光照：Lumen（UE5默认）
  反射：Lumen反射
  阴影：虚拟阴影贴图
  
【平台】Windows
  默认RHI：DirectX 12
```
