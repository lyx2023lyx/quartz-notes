## 12.1 基础知识

### 12.1.1 参数化表面（Parametric Surface）

**核心概念**：使用参数方程显式表述表面上的每一个点

#### 数学定义

对于一个参数化表面，表面上的每一点 $\mathbf{P}$ 可以表示为：

$$
\mathbf{P} = \mathbf{P}(s, t)
$$

其中 $s, t$ 是参数，对于单位圆：

$$
\begin{cases}
x(\theta) = \sin \theta \\
y(\theta) = \cos \theta
\end{cases}
\tag{12.1}
$$

**白话解读**：就像地球仪上的经纬度，给定一对参数值就能确定表面上的一个唯一点。

#### 隐式验证

上述参数方程满足隐式关系：

$$
x^2(\theta) + y^2(\theta) - 1 = 0
\tag{12.2}
$$

**几何意义**：参数化表面上所有点都落在单位圆上。

| 特性 | 描述 | 优缺点 |
|------|------|--------|
| **生成** | 很容易通过参数采样生成表面点 | ✅ 直观、易于光栅化 |
| **内外判断** | 难以判断任意点是否在表面上 | ❌ 需要求解非线性方程 |
| **相交测试** | 光线-表面相交计算复杂 | ❌ 不适合光线追踪 |
| **典型应用** | 三角形网格、NURBS曲面 | 实时渲染的主流选择 |

#### 参数化表面的分类

根据[Goldman, 2009]，表面表述有四种标准方法：

1. **参数化表面**（Parametric Surfaces）：本章讨论
2. **隐式表面**（Implicit Surfaces）：12.1.2节
3. **形变表面**（Deformed Surfaces）：对其他表面执行变形操作
4. **程序化表面**（Procedural Surfaces）：通过程序生成，如分形

### 12.1.2 隐式表面（Implicit Surface）

**核心概念**：将表面定义为满足某个隐式方程 $F(\mathbf{P}) = 0$ 的所有点的集合

#### 数学定义

一个隐式表面定义为点 $\mathbf{P}$ 的集合，满足：

$$
F(\mathbf{P}) = 0 \quad \text{或} \quad \phi(\mathbf{x}) = 0
$$

对于单位圆：

$$
F(x, y) \equiv x^2 + y^2 - 1 = 0
\tag{12.3}
$$

**区域划分**：
- **内部** $\Omega^-$：$\{\mathbf{x} \mid |\mathbf{x}| < 1\}$
- **外部** $\Omega^+$：$\{\mathbf{x} \mid |\mathbf{x}| > 1\}$
- **交界面** $\partial\Omega$：$\{\mathbf{x} \mid |\mathbf{x}| = 1\}$

#### 隐式表面的基本属性

**1. 内外判断**

对于闭合表面，通过函数符号可唯一区分点的位置：

$$
\phi(\mathbf{x}) = 
\begin{cases}
< 0 & \text{如果 } \mathbf{x} \in \Omega^- \text{（内部）} \\
= 0 & \text{如果 } \mathbf{x} \in \partial\Omega \text{（表面上）} \\
> 0 & \text{如果 } \mathbf{x} \in \Omega^+ \text{（外部）}
\end{cases}
\tag{12.4}
$$

**与参数化表面的对比**：
- 隐式表面：✅ 容易判断点是否在表面上或内外
- 参数化表面：❌ 难以判定是否存在参数使 $\mathbf{P} = \mathbf{P}(\theta)$

**2. 点包含性测试方法**

对于参数化表面，常用**环绕数（winding number）**方法：

| 方法 | 原理 | 适用场景 |
|------|------|----------|
| **偶奇规则** | 从点向无穷远发射光线，计数与边界相交次数 | 简单闭合曲线 |
| **非零环绕规则** | 考虑曲线方向，顺时针+1，逆时针-1 | 带方向的几何 |

**3. 表面法线**

隐式表面的法线矢量可由**梯度**给出：

$$
\mathbf{N} = \nabla\phi = \left(\frac{\partial\phi}{\partial x}, \frac{\partial\phi}{\partial y}, \frac{\partial\phi}{\partial z}\right)
\tag{12.5}
$$

**证明**（基于链式法则）：

设 $\mathbf{P}(t) = (x(t), y(t), z(t))$ 是表面上的一条曲线，则：

$$
\phi(x(t), y(t), z(t)) = 0
\tag{12.6}
$$

使用链式法则：

$$
\frac{\partial\phi}{\partial x}\frac{dx}{dt} + \frac{\partial\phi}{\partial y}\frac{dy}{dt} + \frac{\partial\phi}{\partial z}\frac{dz}{dt} = \frac{d\phi}{dt} = 0
\tag{12.7}
$$

即：

$$
\nabla\phi \cdot \mathbf{P}'(t) = 0
\tag{12.8}
$$

**几何意义**：梯度与切线正交，因此梯度方向即为法线方向。

#### 布尔操作（CSG）

隐式函数使构造实体几何（CSG）操作变得非常简单：

| 操作 | 数学表达 | 说明 |
|------|----------|------|
| **并集** | $\phi(x) = \min(\phi_1(x), \phi_2(x))$ | 区域 $\phi_1$ 和 $\phi_2$ 的联合 |
| **交集** | $\phi(x) = \max(\phi_1(x), \phi_2(x))$ | 区域 $\phi_1$ 和 $\phi_2$ 的相交 |
| **补集** | $\phi(x) = -\phi_1(x)$ | 区域 $\phi_1$ 的外部 |
| **差集** | $\phi(x) = \max(\phi_1(x), -\phi_2(x))$ | 从 $\phi_1$ 减去 $\phi_2$ |

**应用场景**：CAD设计、程序化建模

### 12.1.3 有符号距离函数（Signed Distance Function, SDF）

**核心概念**：特殊的隐式函数，其函数值的绝对值等于到最近表面的距离

#### 完整定义

**距离函数**：

对于所有 $\mathbf{x}_I \in \partial\Omega$：

$$
d(\mathbf{x}) = \min_{\mathbf{x}_I}(|\mathbf{x} - \mathbf{x}_I|)
\tag{12.10}
$$

**有符号距离函数**：

满足 $|\phi(\mathbf{x})| = d(\mathbf{x})$ 的隐式函数：

$$
\phi(\mathbf{x}) = 
\begin{cases}
d(\mathbf{x}) = 0 & \mathbf{x} \in \partial\Omega \text{（表面上）} \\
-d(\mathbf{x}) & \mathbf{x} \in \Omega^- \text{（内部，负值）} \\
d(\mathbf{x}) & \mathbf{x} \in \Omega^+ \text{（外部，正值）}
\end{cases}
\tag{12.11}
$$

#### SDF的关键性质

| 性质 | 数学描述 | 实际意义 |
|------|----------|----------|
| **Lipschitz连续** | $\|\phi(\mathbf{x}) - \phi(\mathbf{y})\| \leq \|\mathbf{x} - \mathbf{y}\|$ | 保证球体追踪的安全性 |
| **梯度模为1** | $\|\nabla\phi\| = 1$（几乎处处） | 单位速度传播 |
| **满足Eikonal方程** | $\|\nabla\phi\| = 1$ | 距离场的偏微分方程描述 |
| **内外可区分** | 符号区分内外 | 无需额外计算 |

#### 基本基元的SDF

**球体**（中心在原点，半径$s$）：

```glsl
float sdSphere(vec3 p, float s) {
    return length(p) - s;
}
```

**圆环**（主半径$t.x$，次半径$t.y$）：

```glsl
float sdTorus(vec3 p, vec2 t) {
    vec2 q = vec2(length(p.xz) - t.x, p.y);
    return length(q) - t.y;
}
```

#### SDF的组合操作

| 操作 | GLSL实现 |
|------|----------|
| **并集** | `return min(d1, d2);` |
| **差集** | `return max(-d1, d2);` |
| **交集** | `return max(d1, d2);` |
| **平滑并集** | `smoothmin(d1, d2, k)` |

## 12.2 距离场的计算

### 12.2.1 距离场计算概述

**基本问题**：计算3D空间中所有离散位置到最近物体表面的距离

**计算方法分类**：

```
距离场计算方法
├── 暴力方法（Brute Force）
│   └── 对每个体素向所有方向发射光线
│   └── 示例：Unreal Engine 4 离线生成
│
├── 距离变换（Distance Transform）
│   ├── 切角距离变换（Chamfer DT）
│   └── 矢量距离变换（Vector DT）
│
└── 实时GPU计算
    └── 基于几何着色器/计算着色器
```

### 12.2.2 三角形网格的距离场

**前提条件**：通常需要闭合、有向的二维流形

#### 点到三角形的距离计算

投影点$\mathbf{p}'$落在7个区域之一：

```
         R3
          │
    R6────┼────R7
          │
    ──────┼──────→ 边1
    R2    │    R4
          │
    R5────┼──── 顶点
          │
         R1（三角形内部）
```

| 区域 | 距离计算方式 |
|------|--------------|
| R1 | 到三角形所在平面的距离 |
| R2, R3, R4 | 到对应边的距离 |
| R5, R6, R7 | 到对应顶点的距离 |

#### 符号计算

**方法1：角度加权法线**（Jones et al., 2006）

顶点法线通过相邻面的法线角度加权计算：

$$
\mathbf{N} = \frac{\sum_i \mathbf{n}_i a_i}{\|\sum_i \mathbf{n}_i a_i\|}
$$

其中 $a_i$ 是相邻面在该顶点处的角度。

**方法2：启发式方法**（Unreal Engine 4）

通过光线追踪统计背面击中比例：
- 若背面击中 > 50%，则认为点位于物体"内部"

### 12.2.3 距离变换（Distance Transform）

**核心思想**：先在边界附近精确计算，然后传播到整个空间

#### 切角距离变换（Chamfer Distance Transform）

**距离模板**（Distance Template）：

```
    2b   c   2a   c   2b
     c   b   a    b   c
    2a   a   0    a   2a
     c   b   a    b   c
    2b   c   2a   c   2b

其中：a = 1, b = √2, c = √3
```

**算法流程**：

```
┌─────────────────────────────────────────┐
│  切角距离变换算法                        │
├─────────────────────────────────────────┤
│  输入：二值图像（0=边界，∞=其他）         │
│  输出：距离场                            │
├─────────────────────────────────────────┤
│  1. 初始化：边界点=0，其他=∞              │
│  2. 前向遍历（左上到右下）                │
│     对每个像素，检查已处理的邻居          │
│     更新距离 = min(当前, 邻居+模板值)     │
│  3. 后向遍历（右下到左上）                │
│     同样方式更新                         │
│  4. 返回距离场                           │
└─────────────────────────────────────────┘
```

**精度问题**：距离越远，误差越大

#### 矢量距离变换（Vector Distance Transform）

与切角DT不同，存储的是**指向最近表面的矢量**：

| 特性 | 切角DT | 矢量DT |
|------|--------|--------|
| 存储内容 | 标量距离 | 方向矢量 |
| 精度 | 近似 | 更精确 |
| 计算复杂度 | 低 | 较高 |
| 适用场景 | 快速近似 | 高质量需求 |

**代表算法**：Danielsson (1980), Satherley and Jones

### 12.2.4 距离场的存储

#### 适应性距离场（Adaptive Distance Field, ADF）

**核心思想**：细节驱动的自适应采样

**数据结构**：八叉树（3D）/ 四叉树（2D）

```
┌──────────────────────────────────────────┐
│            ADF 四叉树结构示意             │
├──────────────────────────────────────────┤
│                                          │
│     ┌─────────┬─────────┐               │
│     │  细分   │   细分  │               │
│     │ ┌─┬─┐ │ ┌─┬─┐  │               │
│     │ │高│高│ │ │高│低│  │               │
│     │ │密│密│ │ │密│密│  │               │
│     │ └─┴─┘ │ └─┴─┘  │               │
│     ├─────────┼─────────┤               │
│     │   低    │    低   │               │
│     │   密    │    密   │               │
│     │  (平坦) │  (平坦) │               │
│     └─────────┴─────────┘               │
│                                          │
│  高密 = 曲面细节丰富区域                  │
│  低密 = 平坦区域                          │
└──────────────────────────────────────────┘
```

**细分准则**：比较当前单元与父单元的平均距离差异

**缺点**：
- 平坦区域采样稀疏导致过渡不平滑
- 可能出现斑点（如图12.16天花板顶柱）

#### 完全距离场表述（Complete Distance Field Representation, CDFR）

**核心洞察**：奈奎斯特定理无法捕捉无限频率的表面细节（边角、棱边）

**完全距离定义（CDD）**：

$$
\langle \text{distance}, \langle v_1, v_2, v_3 \rangle, \langle e_1, e_2, e_3 \rangle \rangle
\tag{12.12}
$$

包含：
1. **标量距离值**：体素中心到三角形的距离
2. **三角形顶点列表**：用于计算符号
3. **边列表**：基础三角形描述

**优势**：
- 可提取任意精度要求的距离等量线
- 不依赖线性采样定理

## 12.3 隐式表面渲染

### 12.3.1 表面表述与渲染方法对比

| 渲染方法 | 表面表述 | 核心操作 | 适用场景 |
|----------|----------|----------|----------|
| **光栅化** | 参数化（三角形网格） | 顶点变换+插值 | 实时渲染主流 |
| **光线追踪** | 参数化 + 加速结构 | 光线-三角形相交 | 离线高质量 |
| **光线步进** | 体积密度函数 | 固定步长采样 | 体积云、雾 |
| **球体追踪** | 距离场（SDF） | 自适应步长 | 隐式表面、SDF场景 |

### 12.3.2 光线步进（Ray Marching）

**基本思想**：沿光线方向以固定步长采样

#### 算法描述

光线参数方程：

$$
\mathbf{x} = \mathbf{x}_{\mu_0} + k\Delta\mathbf{x}_\mu
\tag{12.14}
$$

**伪代码**：

```
function RayMarching(ray, t_max, k_max):
    k = 0
    while k < k_max:
        d = f(ray(k * delta_t))
        if d ≤ 0:
            return k * delta_t  // 击中表面
        k = k + 1
    return 0  // 未击中
```

**问题**：
- 步长太大 → 穿透表面、遗漏细节
- 步长太小 → 性能低下

```
    蓝色轮廓 = 物体表面
    绿色点   = 采样点
    
    大步长情形：
    ●━━━━━━━●━━━━━━━━●
            ↑ 穿透表面！
    
    小步长情形：
    ●━●━●━●━●━●━●━●
      计算量大，效率低
```

### 12.3.3 球体追踪（Sphere Tracing）

**核心创新**：使用距离场值作为安全步长

#### 核心概念：未包围体积（Unbounding Volume）

在距离场中的每个点，以该点的距离值为半径的球体内**保证不存在任何物体**。

**算法流程图**：

```
┌─────────────────────────────────────────────────────┐
│                 球体追踪算法                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│   光线方向 →                                        │
│        ↓                                            │
│   ┌─────────────┐   d=3.0                          │
│   │  ○          │   ← 以当前位置为中心，d为半径     │
│   │   ╲         │      球体内无物体，安全！          │
│   │    ╲        │                                  │
│   └─────╫───────┘                                  │
│         ↓ d=2.0                                     │
│   ┌──────╫──────┐                                   │
│   │      ○      │   ← 继续前进                      │
│   │       ╲     │                                  │
│   └────────╫────┘                                  │
│            ↓ d=0.5                                  │
│   ┌─────────○───┐   ← 接近表面，小步长             │
│   │         │   │                                  │
│   └─────────╫───┘                                  │
│             ↓ d≤ε                                   │
│   ══════════●════   ← 击中表面！                    │
│                                                     │
│   ε = 最小距离阈值（如0.001）                       │
└─────────────────────────────────────────────────────┘
```

#### 算法伪代码

```
function SphereTracing(ray, D, epsilon):
    t = 0
    while t < D:
        d = f(ray(t))  // 采样距离场
        if d ≤ epsilon:
            return t     // 收敛到表面
        t = t + d        // 安全步进
    return 0             // 未击中
```

#### 与光线步进的对比

| 特性 | 光线步进 | 球体追踪 |
|------|----------|----------|
| 步长 | 固定 | 自适应（距离值） |
| 安全性 | 可能穿透 | 保证不穿透 |
| 效率 | 低效（均匀采样） | 高效（空白区域大步进） |
| 精度 | 依赖步长 | 依赖收敛阈值 |
| 适用 | 体积云、超纹理 | 隐式表面、SDF |

```
│  每物体距离场（Per-Object DF）                       │
│  ┌─────────────────────────────────────────────┐   │
│  │   高分辨率、小覆盖范围（物体表面附近）          │   │
│  │        ┌─────┐                               │   │
│  │        │高Res│ ← 仅覆盖单个物体               │   │
│  │        └─────┘                               │   │
│  │   用于：精确的表面相交计算                      │   │
│  └─────────────────────────────────────────────┘   │
│                                                     │
│  策略：远→全局DF，近→每物体DF                      │
└─────────────────────────────────────────────────────┘
```

#### 超松弛球体追踪（SOR）

**核心思想**：借鉴超松弛方法的加速原理

**步进增量**：

$$
\delta_i = f(\mathbf{p}_i) \cdot \omega
$$

其中 $\omega \in [1, 2)$ 为外推参数。

**安全回退条件**：

$$
|f(\mathbf{p}_{i-1})| + |f(\mathbf{p}_i)| < \delta_{i-1}
$$

则回退到标准球体追踪：

$$
\mathbf{p}_{\text{fallback}} = \mathbf{p}_i + \mathbf{d} \cdot \delta_{i-1} \cdot (1 - \omega)
\tag{12.16}
$$

**效果**：
- 7次迭代 → 4次迭代（示例）
- 空白区域大幅加速
- 表面附近自动回退保证安全

## 12.4 移位映射（Displacement Mapping）

### 12.4.1 表面结构分类

| 结构类型 | 尺度 | 表述方法 | 示例 |
|----------|------|----------|------|
| **微观结构** | 肉眼不可见 | BRDF统计模型 | 金属表面粗糙度 |
| **中观结构** | 单三角形内 | 移位映射 | 砖缝、浮雕 |
| **宏观结构** | 多三角形 | 几何建模 | 建筑整体形状 |

### 12.4.2 浮雕映射（Relief Mapping）

**技术路线**：基于高度图的GPU光线追踪

#### 算法流程

```
┌─────────────────────────────────────────────────────┐
│              浮雕映射算法                            │
├─────────────────────────────────────────────────────┤
│  输入：深度图、法线图、观察方向                       │
│  输出：移位后的纹理坐标                               │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. 将观察方向VD变换到切线空间                       │
│                                                     │
│  2. 计算光线在深度=1.0处的终点B                      │
│                                                     │
│  3. 线性搜索找到第一个在表面内的点 Ⓝ3                │
│     for i = 0 to N:
│        if depth(sample(A→B, i/N)) > height:
│           found = true
│           break
│                                                     │
│  4. 在 Ⓝ2（上一个外部点）和 Ⓝ3 之间二分搜索         │
│     for i = 0 to M:  // 通常M=8
│        mid = (start + end) / 2
│        if depth(mid) > height:
│           end = mid
│        else:
│           start = mid
│                                                     │
│  5. 返回最终交点作为纹理坐标                         │
└─────────────────────────────────────────────────────┘
```

**问题**：
- 线性搜索需要权衡精度与性能
- 固定步长无法自适应细节

### 12.4.3 基于距离场的移位映射

**改进**：使用球体追踪替代线性+二分搜索

**距离场纹理构建**：

```
1. 创建3D贴图，每个像素存储到最近表面的位移矢量
2. 执行距离变换，将表面位移扩散到整个空间
3. 最终距离值 = |位移矢量|
```

**优势**：
- 自动调整步进尺寸
- 空白区域大步进，细节区域小步进
- 通常8-16次迭代即可

### 12.4.4 壳形映射（Shell Mapping）

**核心概念**：壳形空间 = 基础表面S + 偏移表面$\mathbf{S}_o$

**贝叶斯坐标映射**：

$$
\begin{aligned}
\mathbf{p}_t &= \phi(\mathbf{T}_t, B(\mathbf{T}, \mathbf{p})) \\
\mathbf{p} &= \phi(\mathbf{T}, B(\mathbf{T}_t, \mathbf{p}_t))
\end{aligned}
\tag{12.17}
$$

**解决扭曲问题**：
- 壳形空间直线步进
- 纹理空间距离查询
- 双向映射保持几何一致性

### 12.4.5 一般化移位贴图（GDM）

**核心创新**：增加方向维度

**5D函数定义**：

$$
d_{\text{GDM}}(x, y, z, \theta, \phi) = 
\begin{cases}
d & \text{如果光线与中观结构相交} \\
d_{\text{max}} & \text{如果光线方向无交点} \\
0 & \text{如果点在体积内}
\end{cases}
\tag{12.18}
$$

**渲染优势**：
- 直接查询交点，无需光线步进
- 效率极高
- 线性插值导致放大失真

### 12.4.6 有向距离贴图（DDM）

**核心改进**：结合方向锥的球体追踪 + 二次曲面近似

**数据结构**：

$$
\text{DDM}(V, [\theta_1, \theta_2] \times [\phi_1, \phi_2]) = \min\{\text{dist}(m, v)\}
\tag{12.19}
$$

**压缩方案**：
- 查询表格（粗分辨率）+ 纹理图集（细节块）
- 二次曲面系数存储

**二次曲面方程**：

$$
Ax^2 + By^2 + Cz^2 + Dxy + Eyz + Fzx + Gx + Hy + Iz + J = 0
\tag{12.20}
$$

**双线性曲面**（保证C0连续）：

$$
(1-u)(1-v)\mathbf{p}_1 + u(1-v)\mathbf{p}_2 + uv\mathbf{p}_3 + (1-u)v\mathbf{p}_4
\tag{12.21}
$$

## 12.5 阴影和遮挡

### 12.5.1 阴影的基本定义

**可见性函数分离**：

通过两个假设，反射方程可分离为：

$$
L_o(\mathbf{p}, \omega) = \underbrace{\text{directIllum}(\mathbf{p}, \omega, \$, \bar{L}_c)}_{\text{着色}} \cdot \underbrace{V_{\$}(\mathbf{p})}_{\text{阴影}}
\tag{12.24}
$$

**可见性积分**：

$$
V_{\$}(\mathbf{p}) = \frac{1}{|\$|}\int_{\$} V(\mathbf{p}, \mathbf{q})d\mathbf{q}
\tag{12.25}
$$

**三种阴影区域**：

| 区域 | 可见性 | 光源类型 | 计算方法 |
|------|--------|----------|----------|
| **照亮区** | V = 1 | 点光源/面积光源 | 直接判断 |
| **本影区** | V = 0 | 任意 | 直接判断 |
| **半影区** | 0 < V < 1 | 面积光源 | 软阴影算法 |

### 12.5.2 硬阴影技术

#### 阴影贴图（Shadow Map）

**基本原理**：

```
┌─────────────────────────────────────────────┐
│  阴影贴图算法                                │
├─────────────────────────────────────────────┤
│                                             │
│  Pass 1: 从光源渲染                         │
│  ┌─────────┐                                │
│  │ 光源    │ → 深度缓冲 = Shadow Map        │
│  │ 摄像机  │    （仅存储深度，无颜色）        │
│  └─────────┘                                │
│                                             │
│  Pass 2: 从观察者渲染                        │
│  ┌─────────┐                                │
│  │ 观察    │ → 对每个像素：                  │
│  │ 摄像机  │    1. 变换到光源空间            │
│  │         │    2. 比较深度值                │
│  │         │       pixel_depth > sm_depth ?  │
│  │         │       → 阴影 : 照亮             │
│  └─────────┘                                │
│                                             │
└─────────────────────────────────────────────┘
```

**常见问题**：

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| **阴影瑕疵** | 深度偏差过小，自我遮挡 | 斜率缩放深度偏差 |
| **彼得潘问题** | 深度偏差过大，光照泄漏 | 动态偏差计算 |
| **透视走样** | 分辨率不足 | 级联阴影贴图 |

#### 级联阴影贴图（CSM）

```
┌─────────────────────────────────────────────────────┐
│              级联阴影贴图                            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  观察者                                             │
│     ↘                                              │
│      ┌─────────┬─────────┬─────────┬─────────┐    │
│      │  级联0  │  级联1  │  级联2  │  级联3  │    │
│      │ 高Res   │ 中Res   │ 低Res   │ 超低Res │    │
│      │ 近距离  │ 中距离  │ 远距离  │ 超远距离│    │
│      └─────────┴─────────┴─────────┴─────────┘    │
│                                                     │
│  特点：每个级联单独渲染阴影贴图                      │
│  优势：近处高分辨率，远处低分辨率                     │
│  挑战：级联之间需要平滑过渡                          │
└─────────────────────────────────────────────────────┘
```

### 12.5.3 软阴影技术

#### 基于光线追踪的软阴影

**半影尺寸计算**：

$$
P = \frac{RO}{L}
\tag{12.26}
$$

其中：
- $R$ = 光源尺寸（半径）
- $O$ = 遮挡物到表面距离
- $L$ = 着色点到光源距离

**双缓存方法**：

1. **密度缓存**：记录硬阴影边界（0或1）
2. **距离缓存**：记录到最近遮挡物的距离

**过滤策略**：
- 阴影区内：使用距离缓存值作为核函数窗宽
- 照亮区：交叉搜索寻找影响该像素的阴影像素

### 12.5.4 基于距离场的软阴影

#### UE4的距离场软阴影

**核心思想**：圆锥追踪近似可见性

**算法**：

```
┌─────────────────────────────────────────────────────┐
│           距离场软阴影算法                           │
├─────────────────────────────────────────────────────┤
│                                                     │
│  圆锥从着色点向光源方向延伸                          │
│                                                     │
│      ╲ 光源                                        │
│       ╲                                            │
│        ╲                                           │
│    ┌────╲────┐                                     │
│    │    ○    │ ← 步进采样点                         │
│    │   ╱ ╲   │   记录min(到最近表面距离)            │
│    │  ╱   ╲  │                                     │
│    │ ╱     ╲ │                                     │
│    ├╱───────╲┤                                     │
│    │○       ○│ ← 遮挡物                            │
│    └─────────┘                                     │
│                                                     │
│  可见性近似：                                        │
│                    Distance to Surface              │
│  Shadow = 1 - Min(─────────────────────)            │
│                      Cone Radius                    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**可见性公式**：

$$
\text{Shadow} = 1 - \min\left(\frac{\text{Distance to Surface}}{\text{Cone Radius}}\right)
\tag{12.27}
$$

**距离场AO计算**：

在半球空间发射多个圆锥，统计可见性：

```
AO = 1 - (Σ cone_visibility_i) / N_cones
```

## 🎯 本章总结

**核心观点**：

1. **距离场是一种隐式表面表述**，它将表面编码为空间点到最近表面的有符号距离

2. **球体追踪是渲染距离场的高效算法**，利用距离值作为安全步长，实现自适应光线步进

3. **距离场在实时渲染中的核心价值**：
   - 无需预计算的动态软阴影
   - 高效的环境遮挡计算
   - 几何细节丰富的移位映射

4. **工程权衡**：
   - 精度 vs 内存占用（适应性采样）
   - 质量 vs 性能（分辨率选择）
   - 适用范围（近距离需配合传统阴影贴图）

**学习本章的意义**：

距离场技术代表了实时渲染中可见性计算的一个重要方向。理解这项技术不仅能帮助我们使用UE4等引擎的相关功能，更能启发我们思考如何在GPU上高效地处理空间查询和几何计算问题。

---

> 💡 **关键洞见**：距离场本身不是全局光照技术，而是一种使高效可见性计算成为可能的几何表述方式。全局光照效果（软阴影、AO）只是这种表述的自然应用。
