## 5.1 路径追踪基础

### 5.1.1 从Whitted到分布式光线追踪

#### Whitted光线追踪的局限

| 特性 | Whitted光线追踪 | 真实光照 |
|------|----------------|----------|
| 反射 | 完美镜面反射 | 漫反射、 glossy |
| 折射 | 完美透射 | 散射、吸收 |
| 间接光 | 只处理镜面/折射 | 需要多次反射 |
| 软阴影 | 硬阴影 | 面积光软阴影 |

**核心问题**：无法处理漫反射表面之间的间接光照

#### 分布式光线追踪

Cook等人提出的改进方案：

```
传统光线追踪:              分布式光线追踪:
    │                         │
    │ 单条反射光线             │ 多条反射光线
    ↓                         ↓
   ╱                          ╲╱╱
  ╱                           ╲│╱
 ╱                             │
镜面反射                    glossy反射

从"确定性"采样 → "随机"采样
```

**关键思想**：
- 在光线分布范围内随机采样
- 用多个样本平均近似积分

### 5.1.2 Kajiya路径追踪算法

#### 历史背景

1986年，Jim Kajiya提出**渲染方程**和**路径追踪**算法，统一了光照传输的计算框架。

#### 核心思想

> 从相机出发，递归地随机采样光线路径，用蒙特卡洛方法估计辐射亮度

```
路径追踪算法流程:

相机像素 ──→ 发射主射线 ──→ 与场景相交
                                  │
                                  ▼
                           计算直接光照
                           (采样光源)
                                  │
                                  ▼
                           随机采样下一条射线
                           (按BRDF分布)
                                  │
                                  ▼
                           递归计算间接光照
                                  │
                                  ▼
                           俄罗斯轮盘赌终止
```

#### 算法伪代码

```
function PathTrace(ray, depth):
    if depth > maxDepth:
        return 0
    
    // 1. 光线与场景求交
    hit = Intersect(ray, scene)
    if not hit:
        return Environment(ray)
    
    // 2. 直接光照 (NEE)
    direct = SampleLight(hit.point, hit.normal)
    
    // 3. 间接光照
    // 3.1 按BRDF采样新方向
    newRay = SampleBRDF(hit.point, hit.normal, ray.direction)
    
    // 3.2 计算BRDF值和PDF
    brdf = EvaluateBRDF(...)
    pdf = PDF(...)
    
    // 3.3 递归追踪
    indirect = PathTrace(newRay, depth + 1)
    
    // 3.4 蒙特卡洛估计
    indirect = brdf * indirect * cosθ / pdf
    
    // 4. 俄罗斯轮盘赌
    if RussianRoulette():
        return 0
    
    return direct + indirect / P_survive
```

## 5.2 渲染方程的路径积分形式

### 5.2.1 从积分方程到路径积分

#### 渲染方程回顾

$$
L_o(p, \omega_o) = L_e(p, \omega_o) + \int_\Omega f_r(p, \omega_i, \omega_o) L_i(p, \omega_i) \cos\theta_i d\omega_i
$$

#### 路径空间定义

一条**光路**$\bar{x}$是顶点序列：

$$
\bar{x} = x_0 x_1 x_2 \cdots x_k
$$

其中：
- $x_0$：光源上的点
- $x_k$：相机传感器上的点
- $x_1, \cdots, x_{k-1}$：场景表面上的点

```
路径可视化:

光源        表面A        表面B        相机
  ●─────────●───────────●───────────●
 x₀        x₁          x₂         x₃
 
 路径长度 = 3 (边数)
 路径顶点数 = 4
```

### 5.2.2 路径积分公式

#### 路径贡献值

每条路径的**贡献值**：

$$
C(\bar{x}) = L_e(x_0 \to x_1) \cdot G(x_0 \leftrightarrow x_1) \cdot \prod_{i=1}^{k-1} f_r(x_{i-1} \to x_i \to x_{i+1}) \cdot G(x_i \leftrightarrow x_{i+1}) \cdot W(x_k)
$$

其中：
- $L_e$：光源发射的辐射亮度
- $f_r$：BRDF
- $G$：几何项
- $W$：传感器响应函数

#### 几何项

$$
G(x_i \leftrightarrow x_{i+1}) = V(x_i, x_{i+1}) \cdot \frac{|\cos\theta_i \cos\theta_{i+1}|}{||x_{i+1} - x_i||^2}
$$

其中$V$为可见性函数（0或1）。

### 5.2.3 路径测度

#### 路径空间测度

路径积分形式：

$$
I = \int_\Omega f(\bar{x}) d\mu(\bar{x})
$$

其中$d\mu(\bar{x})$是路径空间上的测度。

#### 路径积分 vs 面积积分

| 形式 | 积分变量 | 特点 |
|------|----------|------|
| **面积积分** | 表面点 | 物理直观 |
| **方向积分** | 立体角 | 渲染方程标准形式 |
| **路径积分** | 光路 | 适合蒙特卡洛分析 |

## 5.3 光线与表面求交

### 5.3.1 光线方程

#### 参数化表示

$$
R(t) = O + t \cdot D, \quad t \in [t_{min}, t_{max}]
$$

其中：
- $O$：光线起点
- $D$：光线方向（归一化）
- $t$：距离参数

### 5.3.2 隐式表面求交

#### 隐式表面定义

表面由方程$F(p) = 0$定义。

#### 求交计算

将光线方程代入隐式方程：

$$
F(O + tD) = 0
$$

解关于$t$的方程。

#### 球面求交示例

球面方程：$||p - c||^2 - r^2 = 0$

代入光线方程：

$$
||O + tD - c||^2 - r^2 = 0
$$

展开：

$$
(D \cdot D)t^2 + 2D \cdot (O-c)t + (O-c) \cdot (O-c) - r^2 = 0
$$

这是关于$t$的二次方程，可用求根公式求解。

```
判别式 Δ = b² - 4ac:

Δ < 0: 无交点
Δ = 0: 相切
Δ > 0: 两个交点 t₁, t₂

取最小的正根作为命中点
```

### 5.3.3 参数化表面求交

#### 参数化表面定义

$$
p(u, v) = (x(u,v), y(u,v), z(u,v))
$$

#### 三角形求交

三角形顶点：$v_0, v_1, v_2$

**Möller-Trumbore算法**：

```
光线与三角形求交:

1. 计算边向量
   e₁ = v₁ - v₀
   e₂ = v₂ - v₀

2. 计算行列式
   pvec = D × e₂
   det = e₁ · pvec

3. 若|det| < ε: 光线与三角形平行，无交点

4. 计算u参数
   tvec = O - v₀
   u = (tvec · pvec) / det
   若u ∉ [0,1]: 无交点

5. 计算v参数  
   qvec = tvec × e₁
   v = (D · qvec) / det
   若v < 0 或 u+v > 1: 无交点

6. 计算t值
   t = (e₂ · qvec) / det
   
7. 返回命中点: O + t·D
```

### 5.3.4 加速结构

#### 问题背景

场景可能有数百万个三角形，每次光线求交都遍历所有三角形代价巨大！

#### 层次包围盒 (BVH)

```
BVH构建:

场景AABB                    根节点
┌─────────────────┐            │
│  □ □  □  □ □ □  │     ┌─────┴─────┐
│   □   □    □    │  →  │           │
│  □ □  □  □ □ □  │   左子树     右子树
└─────────────────┘      │           │
                      ┌──┴──┐     ┌──┴──┐
                     叶子   叶子  叶子   叶子
                     [△△]  [△]   [△△]  [△△]
```

**构建策略**：
- SAH (Surface Area Heuristic)：最小化期望光线遍历代价
- 中轴分割、K-means分割等

#### KD-Tree

```
KD-Tree构建 (交替沿xyz轴分割):

┌─────────────────┐
│  □ □ │ □  □ □  │  ← 沿x轴分割
│   □  │ □    □  │
│  □ □ │ □  □ □  │
├──────┼─────────┤
│  □ □ │ □  □ □  │  ← 子空间沿y轴分割
│   □  │ □    □  │
└──────┴─────────┘
```

#### BVH vs KD-Tree

| 特性 | BVH | KD-Tree |
|------|-----|---------|
| 构建速度 | 快 | 较慢 |
| 遍历速度 | 快 | 很快 |
| 动态更新 | 较容易 | 较困难 |
| 内存占用 | 较少 | 较多 |
| 现代使用 | 主流 | 部分应用 |

## 5.4 路径采样策略

### 5.4.1 BRDF采样

#### 原理

按照BRDF的分布采样反射方向：

$$
\omega_i \sim f_r(\omega_i, \omega_o) \cos\theta_i
$$

#### 常用BRDF采样

| BRDF类型 | 采样方法 | PDF |
|----------|----------|-----|
| **Lambertian** | 余弦加权半球采样 | $p(\omega) = \cos\theta / \pi$ |
| **Phong** | 指数加权反射方向 | $p(\omega) \propto (\omega \cdot r)^n$ |
| **GGX** | 可见法线分布采样 | 复杂解析式 |

#### 余弦加权半球采样

```
算法:

1. 生成均匀随机数 ξ₁, ξ₂ ∈ [0,1]

2. 计算球坐标:
   θ = arccos(√(1-ξ₁))   // 极角
   φ = 2πξ₂              // 方位角

3. 转为笛卡尔坐标:
   x = sinθ cosφ
   y = sinθ sinφ  
   z = cosθ       ← 法线方向

PDF: p(θ,φ) = cosθ/π
```

### 5.4.2 Next Event Estimation (NEE)

#### 核心思想

> 在每次反弹时，直接采样光源计算直接光照

```
传统路径采样:           NEE:
    │                     │
    │ 随机采样            ├─→ 采样光源 (直接)
    │                     │
    ▼                     ├─→ 随机采样BRDF (间接)
   可能很久才              │
   碰到光源               ▼
                         更快收敛
```

#### 直接光照估计

对于面积光源$LS$：

$$
L_{direct} = \frac{1}{N} \sum_{i=1}^{N} \frac{L_e(x_i) f_r(\omega_i) \cos\theta_i \cos\theta'}{p(x_i) ||x_i - p||^2} V(p, x_i)
$$

#### 光源采样策略

| 策略 | 适用场景 |
|------|----------|
| **均匀采样** | 小光源、简单场景 |
| **按功率采样** | 多光源场景 |
| **按立体角采样** | 近光源 |
| **MIS组合** | 通用场景 |

### 5.4.3 多重重要性采样 (MIS)

#### 问题场景

```
纯BRDF采样:            纯光源采样:

光源小+镜面BRDF        光源大+漫反射BRDF
    │                       │
    │ ╲                     │
    │  ╲                    │    ╱╲
    │   ╲                   │   ╱  ╲
    │    ╲                  │  ╱    ╲
    ▼     ↘光源             │ ╱      ↘光源
   很难采样到               ▼
   光源                    很难采样到
                           高光区域
```

单一采样策略难以应对所有情况！

#### MIS权重计算

平衡启发式：

$$
w_{BRDF}(x) = \frac{p_{BRDF}(x)}{p_{BRDF}(x) + p_{light}(x)}
$$

$$
w_{light}(x) = \frac{p_{light}(x)}{p_{BRDF}(x) + p_{light}(x)}
$$

#### 组合估计

$$
L = \frac{w_{BRDF} \cdot L_{BRDF}}{p_{BRDF}} + \frac{w_{light} \cdot L_{light}}{p_{light}}
$$

### 5.4.4 俄罗斯轮盘赌 (Russian Roulette)

#### 问题

路径长度可能无限，需要有效终止策略。

#### 算法

```
俄罗斯轮盘赌:

设当前路径贡献为C，生存概率为P

1. 生成随机数 ξ ∈ [0,1]

2. 若 ξ > P:
   // 路径终止
   return 0

3. 否则:
   // 路径继续，但权重调整
   return C / P
```

#### 生存概率选择

| 策略 | 公式 | 特点 |
|------|------|------|
| **固定概率** | $P = 0.9$ | 简单但非最优 |
| **按吞吐率** | $P \propto \text{albedo}$ | 自适应 |
| **按最大分量** | $P = \max(R,G,B)$ | 常用 |

## 5.5 Disney Hyperion渲染器

### 5.5.1 架构概述

#### 设计目标

> 工业级电影渲染器，处理大规模复杂场景

| 特性 | 说明 |

| **延迟几何加载** | 处理超大规模场景 |

### 5.5.2 光线排序

#### 问题

传统路径追踪：
```
光线R₁ → 三角形A → 三角形B → 三角形C
光线R₂ → 三角形X → 三角形Y → 三角形Z  
光线R₃ → 三角形A → 三角形B → 三角形C
         ↓
    缓存不断失效！
```

#### 解决方案

> 按材质/几何对光线进行排序，批量处理

```
排序前:              排序后:

R₁→A,B,C            R₁→A,B,C
R₂→X,Y,Z     →     R₃→A,B,C  (批次1: A,B,C)
R₃→A,B,C            R₂→X,Y,Z  (批次2: X,Y,Z)

缓存友好！
```

### 5.5.3 路径正则化

#### 问题

高粗糙度表面上的高光反射可能产生噪点。

#### 正则化策略

```
路径追踪中的方差来源:

┌─────────────────┐
│ 高光反射采样    │ → 限制最大光线差异
│ (高方差)        │
├─────────────────┤
│ 直接光源采样    │ → 使用MIS  
│ (中低方差)      │
├─────────────────┤
│ 漫反射采样      │ → 标准处理
│ (低方差)        │
└─────────────────┘
```

### 5.5.4 自适应采样

#### 原理

根据局部方差动态调整采样数：

```
像素方差估计:              采样分配:

┌───┬───┬───┐             ┌───┬───┬───┐
│低 │高 │低 │             │1sp│8sp│1sp│
├───┼───┼───┤    →       ├───┼───┼───┤
│低 │高 │高 │             │1sp│8sp│8sp│
├───┼───┼───┤             ├───┼───┼───┤
│低 │低 │低 │             │1sp│1sp│1sp│
└───┴───┴───┘             └───┴───┴───┘

sp = samples per pixel
```
