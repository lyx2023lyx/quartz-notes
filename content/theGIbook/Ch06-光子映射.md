## 6.1 光子映射原理

### 6.1.1 路径追踪的局限

#### 困难路径问题

| 现象 | 问题描述 | 为什么困难 |
|------|----------|------------|
| **焦散(Caustics)** | 光经镜面/折射聚焦 | 光源→镜面→漫反射，路径概率极低 |
| **镜面-漫反射-镜面(SDS)** | 光穿过镜面/水面 | 路径追踪难以采样 |
| **强间接光照** | 小窗室内场景 | 直接光源采样失效 |

#### 焦散示例

```
路径追踪采样焦散:

         光源
           ●
           │
           │ 反射/折射光线
           ▼
    ~~~~~~~~~~~~~~~ 水面
           │
           │ 需要精确命中
           ▼
          ╱ ╲    盆底漫反射
         ╱   ╲
        ╱     ╲
       ╱       ╲
      ▔▔▔▔▔▔▔▔▔▔▔
      
问题：从相机出发的随机采样
     很难"恰好"经过反射点到达光源
```

### 6.1.2 双向方法

#### 核心思想

> **从光源和相机双向工作**：光子从光源出发存储能量，渲染时查询附近光子估计光照

```
双向方法概念:

光源端 ─────────────────────────→ 相机端
   │                                    │
   ▼                                    ▼
发射光子                             发射视线
   │                                    │
   ▼                                    ▼
存储光子图                           光线追踪
   │                                    │
   └────────────┬───────────────────────┘
                │
                ▼
         渲染时查询光子图
         估计辐射亮度
```

#### 与路径追踪对比

| 特性 | 路径追踪 | 光子映射 |
|------|----------|----------|
| 方向 | 相机→光源 | 光源+相机双向 |
| 焦散处理 | 困难 | 高效 |
| 内存使用 | 低 | 高(存储光子) |
| 偏差 | 无偏 | 有偏(但一致) |
| 渐进收敛 | 直接 | 需要再发射 |

## 6.2 光子追踪

### 6.2.1 光子发射

#### 光源采样

从光源表面采样发射点：

```
光子发射过程:

光源表面                    光子
┌─────────┐                
│         │     ──────→     ●──────→
│    ●    │    采样位置    (位置,方向,能量)
│   ╱│╲   │                
│  ╱ │ ╲  │    ──────→     ●──────→
└─────────┘                

1. 采样发射位置 (按功率密度)
2. 采样发射方向 (按光源分布)
3. 分配初始能量
```

#### 发射功率计算

每个光子的功率：

$$
\Phi_{photon} = \frac{\Phi_{light}}{N_{photons}}
$$

对于多光源：

$$
\Phi_{photon}^i = \frac{\Phi_{light}^i}{N_{light} \cdot p_i}
$$

其中$p_i$是光源被采样的概率。

### 6.2.2 光子传播

#### 传播过程

```
光子追踪流程:

光源发射光子
     │
     ▼
光线与场景求交
     │
     ├──→ 无交点 ──→ 丢弃/环境光
     │
     ▼
存储光子 (漫反射表面)
     │
     ▼
决定交互类型 (Russian Roulette)
     │
     ├──→ 吸收 ──→ 终止
     │
     ├──→ 漫反射 ──→ 继续追踪
     │
     └──→ 镜面/折射 ──→ 继续追踪
```

#### 表面交互

使用Russian Roulette决定交互：

| 事件 | 概率 | 处理方式 |
|------|------|----------|
| 吸收 | $P_{abs} = 1 - \text{albedo}$ | 终止 |
| 漫反射 | $P_{diff} = \text{albedo} \cdot (1 - \text{specular})$ | 漫反射方向 |
| 镜面/折射 | $P_{spec} = \text{albedo} \cdot \text{specular}$ | 反射/折射方向 |

### 6.2.3 光子存储

#### 存储条件

只在**漫反射表面**存储光子：

```
存储决策:

光子到达表面
     │
     ▼
表面类型?
     │
     ├──→ 镜面/折射 ──→ 不存储，继续追踪
     │
     └──→ 漫反射 ──→ 存储光子，继续追踪
     
为什么? 漫反射表面光照变化平滑，
       适合用密度估计近似
```

#### 存储信息

每个光子存储：

```c
struct Photon {
    Point3 position;      // 位置
    Vector3 direction;    // 入射方向
    Color power;          // 能量
    // 可选: 表面法线、 flags等
};
```

## 6.3 光子图构建

### 6.3.1 KD-Tree结构

#### 为什么用KD-Tree

> 高效的空间查询结构，适合最近邻搜索

```
KD-Tree构建 (2D示例):

Level 0: x轴分割          Level 1: y轴分割
    │                           │
    ▼                           ▼
┌─────┬─────┐              ┌──┬─┴─┬──┐
│     │     │              │  │   │  │
│  A  │  B  │      →       │A │ C │B │
│     │     │              │  │   │  │
└─────┴─────┘              └──┴───┴──┘
                          D  E   F  G
                          
交替沿各维度分割，直到叶子节点包含少量点
```

#### 构建复杂度

- **构建**：$O(N \log N)$
- **查询**：$O(\log N)$ 平均
- **空间**：$O(N)$

### 6.3.2 多遍光子图

#### 全局光子图 vs 焦散光子图

| 类型 | 用途 | 存储位置 |
|------|------|----------|
| **全局光子图** | 间接漫反射光照 | 所有漫反射表面 |
| **焦散光子图** | 焦散效果 | 接收焦散的表面 |
| **体光子图** | 参与介质 | 体积内部 |

#### 分离焦散的好处

```
光子分类存储:

光源                          光源
  │                             │
  ├──→ 直接照射 ──→ 不存         ├──→ 镜面反射 ──→
  │                             │       │
  ├──→ 漫反射 ──→ 全局光子图      │       ▼
  │                             │   焦散光子
  └──→ 镜面→漫反射 ──→ 焦散光子图 │       │
                                        ▼
                                    焦散光子图
                                    
分离原因: 焦散光子通常更密集、
         需要不同渲染参数
```

## 6.4 辐射亮度估计

### 6.4.1 密度估计原理

#### 核心思想

> 用附近光子的密度估计该点的辐射亮度

```
密度估计概念:

查询点x的亮度 ≈ 附近光子的总能量 / 搜索区域面积

                    查询点 ●
                          │
        ┌─────────────────┼─────────────────┐
        │   ○  ○          │      ○          │  ○ = 光子
        │        ○        ○                 │
        │    ○      ○     │    ○            │
        │                 │                 │
        └─────────────────┴─────────────────┘
              搜索半径r内的光子
```

#### 数学公式

反射辐射亮度估计：

$$
L_r(x, \omega_o) \approx \sum_{i=1}^{N} f_r(x, \omega_i, \omega_o) \frac{\Delta\Phi_i(x, \omega_i)}{\pi r^2}
$$

其中：
- $N$：搜索到的光子数
- $r$：搜索半径
- $\Delta\Phi_i$：第$i$个光子的能量
- $\pi r^2$：搜索球面积

### 6.4.2 最近邻搜索 (K-NN)

#### 算法

```
K-NN搜索:

输入: 查询点x, 光子数k
输出: k个最近光子及其距离

1. 优先队列Q ← 空 (最大堆)
2. KDTreeSearch(root, x, Q)
   
function KDTreeSearch(node, x, Q):
    if node是叶子:
        for 每个光子p in node:
            if Q.size < k:
                Q.push(p, distance(x,p))
            else if distance(x,p) < Q.max_distance:
                Q.pop()
                Q.push(p, distance(x,p))
    else:
        // 决定搜索顺序
        if x[axis] < node.split:
            KDTreeSearch(node.left, x, Q)
            if 需要搜索右子树:
                KDTreeSearch(node.right, x, Q)
        else:
            KDTreeSearch(node.right, x, Q)
            if 需要搜索左子树:
                KDTreeSearch(node.left, x, Q)
```

#### 特性

- **固定光子数**：$k$固定，半径$r$变化
- **偏差**：搜索半径越大，估计越平滑
- **方差**：$k$越小，方差越大

### 6.4.3 范围搜索 (Range Search)

#### 算法

```
范围搜索:

输入: 查询点x, 固定半径r
输出: 半径r内所有光子

function RangeSearch(node, x, r, result):
    if node是叶子:
        for 每个光子p in node:
            if distance(x,p) ≤ r:
                result.add(p)
    else:
        if x[axis] - r < node.split:
            RangeSearch(node.left, x, r, result)
        if x[axis] + r ≥ node.split:
            RangeSearch(node.right, x, r, result)
```

### 6.4.4 核密度估计 (KDE)

#### 问题

简单密度估计：
- 搜索球内所有光子权重相同
- 边界处不连续

#### 核函数

使用**核函数**对光子加权：

$$
L_r(x, \omega_o) \approx \sum_{i=1}^{N} f_r(x, \omega_i, \omega_o) \frac{\Delta\Phi_i(x, \omega_i) K(||x - x_i||)}{\int K(||x - x'||) dx'}
$$

#### 常用核函数

| 核函数 | 公式 | 特点 |
|--------|------|------|
| **均匀** | $K(d) = 1$ | 简单，不连续 |
| **圆锥** | $K(d) = 1 - d/r$ | 线性递减，C0连续 |
| **高斯** | $K(d) = e^{-d^2/2\sigma^2}$ | 平滑，无限支撑 |
| **Epanechnikov** | $K(d) = 1 - (d/r)^2$ | 最优收敛率 |

#### 圆锥核示例

```
圆锥核权重分布:

权重
  1 │    ╱╲
    │   ╱  ╲
    │  ╱    ╲
    │ ╱      ╲
    │╱        ╲
    └──────────→ 距离
    0    r/2    r
    
中心光子权重最大
边缘光子权重递减
```

### 6.4.5 渐进光子映射 (PPM)

#### 问题

传统光子映射：
- 固定光子数，有偏估计
- 减少偏差需要增加光子 → 内存爆炸

#### 渐进思想

```
PPM迭代过程:

第1轮: 发射N个光子      第2轮: 再发射N个光子
       ↓                      ↓
    ┌─────┐                ┌─────┐
    │ ●●● │    +           │ ○○○ │
    │ ●●● │ ─────────→     │ ●●● │
    │ ●●● │                │ ○○○ │
    └─────┘                └─────┘
    估计值L₁               估计值L₂
    
最终估计: (L₁ + L₂ + ... + Lₙ) / n
```

#### 收敛性

- 每轮减少搜索半径
- 估计值是**无偏**的
- 方差随轮数增加而减小

## 6.5 参与介质渲染

### 6.5.1 体渲染方程

#### 参与介质中的光传输

```
参与介质中的光传输:

入射光 Lᵢ                  出射光 Lₒ
  │                          ▲
  │    吸收 ──→ (减弱)        │
  │    散射 ──→ (改变方向)    │
  │    发射 ──→ (增强)        │
  ▼                          │
●────────────────────────────●
        介质体积
```

#### 体渲染方程

$$
\frac{dL(x, \omega)}{ds} = -\sigma_t L(x, \omega) + \sigma_a L_e(x, \omega) + \sigma_s \int_{\Omega} p(x, \omega', \omega) L(x, \omega') d\omega'
$$

其中：
- $\sigma_t = \sigma_a + \sigma_s$：消光系数
- $\sigma_a$：吸收系数
- $\sigma_s$：散射系数
- $p$：相位函数

### 6.5.2 体光子映射

#### 体光子追踪

```
体光子追踪:

光源发射光子
     │
     ▼
进入体积介质
     │
     ├──→ 采样自由程 ──→ 决定碰撞位置
     │
     ▼
碰撞点决策 (RR)
     │
     ├──→ 吸收 ──→ 存储光子，终止
     │
     └──→ 散射 ──→ 采样新方向，继续
```

#### 体光子存储

在体积内部存储光子，记录：

```c
struct VolumePhoton {
    Point3 position;      // 碰撞位置
    Vector3 direction;    // 入射方向
    Color power;          // 能量
    float scatter_count;  // 散射次数
};
```

#### 体辐射亮度估计

在体积点$x$处估计：

$$
L(x, \omega) \approx \frac{1}{\frac{4}{3}\pi r^3} \sum_{i=1}^{N} \Delta\Phi_i \cdot p(x, \omega_i, \omega)
$$

注意：体积是3D，搜索区域是球体，分母为球体积$\frac{4}{3}\pi r^3$。

### 6.5.3 光束追踪 (Beam Tracing)

#### 问题

体光子映射在**稀疏区域**方差大。

#### 光束追踪思想

```
光束追踪:

传统: 点-点查询          光束追踪: 光束-体积查询

   ●──────→                ═══════════════
      光子                    光束
                            │
      ●                     ▼
   查询点              ┌──────────┐
                      │   体素   │  ← 累加通过光束的能量
                      └──────────┘
```

存储**光束**而非点，渲染时计算视线与光束的交点。

## 常见问题FAQ

**Q1: 光子映射为什么是有偏的？**
> 因为使用了密度估计，搜索半径固定导致估计有偏差。但随着光子数增加，偏差趋于0（一致）。

**Q2: 存储多少光子合适？**
> 取决于场景复杂度和内存限制。典型值：数十万到数千万。

**Q3: 搜索半径如何选择？**
> 需要在偏差（大半径）和方差（小半径）之间权衡。通常自适应选择。

**Q4: 光子映射能处理镜面表面吗？**
> 不直接存储镜面表面的光子，但镜面反射后的漫反射表面可以接收焦散光子。
