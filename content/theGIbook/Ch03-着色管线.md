| 概念 | 定义 | 本章关注点 |
|------|------|-----------|

## 3.1 着色技术基础

### 渲染方程的简化

原始渲染方程：

$$
L_o(p,\mathbf{v}) = L_e(p,\mathbf{v}) + \int_\Omega f(\mathbf{l},\mathbf{v}) \otimes L_i(p,\mathbf{l}) \cos{\theta_i} \, d\omega_i
$$

**问题**：这个方程不适合直接放入GPU着色器，因为：
1. 包含积分，需要迭代求解
2. $L_i$ 未知（需要追踪光路）
3. 计算量太大

### 实用着色方程

实时渲染中，将渲染方程分解为可直接计算的参数形式：

$$
L_o(\mathbf{v}) = \sum_{k=1}^{n} f_{shade}(E_{L_k}, \mathbf{l}_k, \mathbf{v}, \mathbf{n}, \mathbf{c}_{diff}, \mathbf{c}_{spec}, m)
$$

**白话解读**：
- **$\sum$**：所有光源贡献的累加
- **$E_{L_k}$**：第k个光源的辐射照度
- **$\mathbf{l}_k$**：入射光方向
- **$\mathbf{v}$**：观察方向
- **$\mathbf{n}$**：表面法线
- **$\mathbf{c}_{diff}, \mathbf{c}_{spec}$**：漫反射/高光反射率
- **$m$**：粗糙度/高光扩散系数

**光源辐射照度计算**：

$$
E = I \cdot \cos{\theta} \cdot f_{dist}(r)
$$

其中：
- $I$：光源辐射强度
- $\cos{\theta}$：入射角余弦
- $f_{dist}(r)$：距离衰减函数

### 前向着色（Forward Shading）的问题

```
传统前向着色流程：
┌─────────┐    ┌─────────┐    ┌─────────┐
│ 顶点数据 │───▶│ 顶点着色器│───▶│ 光栅化  │
└─────────┘    └─────────┘    └────┬────┘
                                    │
                                    ▼
                              ┌─────────┐
                              │片元着色器│ ◀── 对所有光源循环计算
                              │ (所有光源)│
                              └────┬────┘
                                   │
                                   ▼
                              ┌─────────┐
                              │  输出   │
                              └─────────┘

问题：
1. 每个像素对所有光源计算（即使是不可见像素）
2. 光源数量增加 → 计算量线性增长
3. 复杂材质开销大
```

## 3.2 延迟着色（Deferred Shading）

### 核心思想

**延迟**：将光照计算推迟到可见性确定之后。

```
延迟着色流程：

几何阶段（Geometry Pass）：
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────────────────┐
│ 顶点数据 │───▶│ 顶点着色器│───▶│ 光栅化  │───▶│ 写入G-Buffer        │
└─────────┘    └─────────┘    └─────────┘    │ ├─ 位置              │
                                              │ ├─ 法线              │
                                              │ ├─ 漫反射颜色        │
                                              │ └─ 材质参数          │
                                              └─────────────────────┘

光照阶段（Lighting Pass）：
┌─────────────────────┐    ┌─────────────┐    ┌─────────┐
│ 读取G-Buffer        │───▶│ 光照着色器   │───▶│ 最终图像 │
│ (仅可见像素的数据)   │    │ (逐光源计算) │    └─────────┘
└─────────────────────┘    └─────────────┘
```

### G-Buffer结构

```
G-Buffer通常包含以下通道：

┌─────────────────────────────────────────────────────────────┐
│                    G-Buffer Layout                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  RT0: [R] [G] [B] [A]  ──▶ 漫反射颜色 (Diffuse Color)       │
│       8bit 8bit 8bit 8bit                                    │
│                                                              │
│  RT1: [R] [G] [B] [A]  ──▶ 法线 (Normal) XYZ + 粗糙度       │
│       10bit 10bit 10bit 2bit                                 │
│                                                              │
│  RT2: [R] [G] [B] [A]  ──▶ 自发光/高光颜色 + 金属度         │
│                                                              │
│  RT3: [R] [G] [B] [A]  ──▶ 深度 (Depth) + 其他              │
│                                                              │
│  深度缓冲: 深度值 (Z)                                       │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**优势**：
1. 光照计算只对可见像素执行
2. 光源与几何解耦，可独立优化
3. 材质复杂度不影响几何阶段性能

**劣势**：
1. 内存带宽要求高（需要读写G-Buffer）
2. 半透明物体处理复杂
3. 抗锯齿困难（丢失几何信息）
4. 无法处理复杂材质混合

### Deferred Lighting vs Deferred Shading

| 特性 | Deferred Shading | Deferred Lighting |
|------|------------------|-------------------|
| G-Buffer内容 | 完整材质参数 | 仅几何信息 |
| 光照计算后 | 直接输出颜色 | 再与漫反射纹理结合 |
| 内存占用 | 大 | 较小 |
| 灵活性 | 材质变化受限 | 支持更多材质类型 |

## 3.3 光源管理

### 光源数量问题

前向/延迟着色都面临光源数量增加的性能问题：

| 场景类型 | 典型光源数量 | 问题 |
|---------|-------------|------|
| 简单场景 | 1-10个 | 可接受 |
| 城市场景 | 100-1000个 | 需要优化 |
| 复杂室内 | 数千个 | 必须使用分块技术 |

### 分块着色（Tiled Shading）

**核心思想**：将屏幕分成小块（Tiles），只计算影响每块的光源。

```
分块着色原理：

屏幕分割：                         光源剔除：
┌────┬────┬────┐                  ┌────┬────┬────┐
│ T0 │ T1 │ T2 │                  │L0,1│L1  │L1,2│
├────┼────┼────┤                  ├────┼────┼────┤
│ T3 │ T4 │ T5 │  ──▶ 深度范围    │L0  │L0,4│L2,4│
├────┼────┼────┤    + 视锥体      ├────┼────┼────┤
│ T6 │ T7 │ T8 │    ──▶ 光源列表  │L3  │L3,4│L4  │
└────┴────┴────┘                  └────┴────┴────┘

每个Tile只处理影响它的光源
```

**计算流程**：
1. 根据深度缓冲计算每个Tile的3D包围盒
2. 测试哪些光源与该包围盒相交
3. 为每个Tile生成光源列表
4. 着色时只遍历该Tile的光源列表

### 分簇着色（Clustered Shading）

**核心思想**：在分块基础上增加深度维度，形成3D分簇。

```
分簇 vs 分块：

分块（2D）：                      分簇（3D）：
┌────┬────┐                      
│ T0 │ T1 │  ← 屏幕XY平面        ┌────┬────┐
├────┼────┤                      │C00 │C01 │
│ T2 │ T3 │                      │C10 │C11 │  ← XY平面 + Z深度
└────┴────┘                      │C20 │C21 │
                                 └────┴────┘
                                  ↑    ↑
                                 Z0   Z1  ← 对数深度分簇
```

**分簇构建**：
- X、Y：屏幕分块（如32×32像素）
- Z：对数深度分簇（解决远近精度问题）

**优势**：
1. 更好的光源剔除精度
2. 支持透明物体
3. 远近光源都能高效处理
4. 适合大规模光源场景

## 3.4 着色器管理

### Uber Shader vs 专用Shader

| 方案 | 特点 | 适用场景 |
|------|------|---------|
| **Uber Shader** | 一个大shader包含所有功能，用宏开关控制 | 简单项目，快速迭代 |
| **专用Shader** | 每个材质类型独立的shader | 大型项目，性能敏感 |
| **Shader变体** | 编译时生成多个变体，运行时选择 | 现代引擎主流方案 |

### 材质系统架构

```
现代引擎材质系统：

┌─────────────────────────────────────────────────────────────┐
│                        材质实例                             │
│                    (Material Instance)                      │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  父材质 (Master Material)                                    │
│       │                                                      │
│       ├── 材质蓝图/Shader Graph                              │
│       │      ├─ 输入参数定义                                 │
│       │      ├─ 节点网络                                     │
│       │      └─ 输出连接到材质属性                           │
│       │                                                      │
│       └── 着色器变体 (Shader Variants)                       │
│              ├─ 启用/禁用特征                                │
```

## 3.5 抗锯齿技术

### 走样问题回顾

| 走样类型 | 原因 | 解决方案 |
|---------|------|---------|
| **几何走样** | 三角形边缘不连续 | MSAA、超采样 |
| **着色走样** | 高光/法线高频变化 | 预过滤、TAA |
| **纹理走样** | 纹理缩小/放大 | Mipmap、各向异性过滤 |
| **时间走样** | 运动物体采样不足 | 运动模糊、TAA |

### 多重采样抗锯齿（MSAA）

**原理**：
- 每个像素多个子采样点（如4x MSAA = 4个子采样点）
- 只计算一次片元着色器（在像素中心）
- 深度/模板测试在每个子采样点执行

```
MSAA采样模式示例：

2x MSAA：              4x MSAA：              8x MSAA：
┌─────┐               ┌─────┐               ┌─────┐
│ ●   │               │ ●   │               │ ● ● │
│   ● │               │   ● │               │●   ●│
└─────┘               │ ●   │               │●   ●│
                      └─────┘               └─────┘
                      ● = 子采样点位置
```

**优点**：
- 几何边缘平滑效果好
- 相比超采样性能开销小

**缺点**：
- 对着色走样无效
- 内存占用增加（深度/颜色缓冲×采样数）

### 时间性抗锯齿（TAA）

**原理**：利用时间累积，多帧采样后混合。

```
TAA工作流程：

当前帧：                历史帧：               混合结果：
┌─────┐                ┌─────┐               ┌─────┐
│ ○   │      +         │   ● │      =        │ ◉   │
└─────┘                └─────┘               └─────┘
 ○ = 当前采样点           ● = 历史采样点         ◉ = 混合后

每帧采样位置偏移（如Halton序列）
多帧累积实现超采样效果
```

**关键问题**：
1. **鬼影（Ghosting）**：历史帧中物体已移动
   - 解决方案：运动向量 + 深度测试

2. **闪烁（Flickering）**：快速变化像素
   - 解决方案：色调映射后混合、钳制历史颜色

### 形态学抗锯齿（MLAA / FXAA / SMAA）

**原理**：后处理阶段检测边缘，对边缘像素进行混合。

**流程**：
1. 检测颜色/深度不连续边缘
2. 计算边缘方向（水平/垂直/斜向）
3. 根据边缘形状混合相邻像素

**对比**：

| 技术 | 原理 | 质量 | 性能 |
|------|------|------|------|
| **FXAA** | 快速近似边缘检测 | 低-中 | 极高 |
| **MLAA** | 形态学边缘分析 | 中 | 高 |
| **SMAA** | 改进的形态学分析 | 高 | 中高 |
| **TAA** | 时间累积 | 很高 | 中 |
| **MSAA** | 几何超采样 | 高 | 中-低 |
