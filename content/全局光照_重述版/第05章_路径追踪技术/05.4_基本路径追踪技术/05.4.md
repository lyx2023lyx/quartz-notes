# 5.4 基本路径追踪技术

## 核心总结
路径追踪就是从相机发射光线，随机选择反射方向，一路追踪到光源，重复多次取平均。关键是正确采样、计算贡献、处理终止。

---

## 路径追踪的基本流程

### 伪代码

```
对于每个像素：
    颜色 = 0
    重复 N 次（采样N条路径）：
        光线 = 从相机穿过像素发射
        路径贡献 = Trace(光线)
        颜色 += 路径贡献
    像素颜色 = 颜色 / N

函数 Trace(光线)：
    如果 光线没击中任何物体：
        返回 环境光颜色
    
    击中点 = 光线与场景最近的交点
    
    如果 击中点是光源：
        返回 光源亮度
    
    // 俄罗斯轮盘赌：决定是否继续追踪
    如果 随机数 > 存活概率：
        返回 0  // 终止这条路径
    
    // 选择下一个反射方向
    新方向 = 根据BRDF随机采样方向
    
    // 计算这条路径的贡献
    新光线 = 从击中点往新方向发射
    后续贡献 = Trace(新光线)
    
    贡献 = BRDF值 × cosθ × 后续贡献 / (存活概率 × PDF)
    
    返回 贡献
```

---

## 关键步骤详解

### 1. 相机光线生成

**怎么做**：
- 相机位置 + 像素位置 → 光线方向
- 可以加上随机偏移（抗锯齿）

**公式**：
```
光线起点 = 相机位置
光线方向 = 归一化(像素位置 - 相机位置)
```

**技巧**：
- 在像素内随机采样（避免锯齿）
- 模拟景深（镜头不是点）
- 模拟运动模糊（时间采样）

---

### 2. 光线-场景相交

**目标**：
找到光线和场景中最近的交点

**方法**：
- 遍历所有三角形（太慢）
- 用加速结构（BVH、KD树、八叉树）

**加速结构原理**：
1. 把场景分成大盒子
2. 光线先和盒子测交点
3. 只有击中盒子才检查里面的物体
4. 递归细分，直到叶子节点

**现代GPU**：
- NVIDIA RT Core专门做这个
- 比CPU快几十倍

---

### 3. BRDF采样

**问题**：
- 击中了表面，往哪反射？
- 漫反射？镜面反射？还是中间？

**方法**：
根据材质类型选择采样策略

#### 漫反射表面

**特点**：
- 光往各个方向均匀散射
- 像白墙、粗糙木头

**采样**：
- 在半球内均匀随机选一个方向
- 或者按cosθ加权（重要性采样）

**PDF**：
```
PDF = cosθ / π
```

#### 镜面反射表面

**特点**：
- 光往特定方向反射
- 像镜子、光滑金属

**采样**：
- 直接算反射方向
- 不需要随机

**PDF**：
```
PDF = 1  // 确定性的
```

#### 光泽表面（中间状态）

**特点**：
- 既不是纯漫反射，也不是纯镜面
- 像光滑塑料、磨砂金属

**采样**：
- 按粗糙度混合两种策略
- 或者直接用微表面模型采样

---

### 4. 计算路径贡献

**公式**：
```
贡献 = BRDF × cosθ × 后续贡献 / PDF
```

**各项解释**：
- **BRDF**：材质的反射特性
- **cosθ**：入射角的影响（斜着照就没那么亮）
- **后续贡献**：从反射方向追踪回来的光
- **PDF**：采样概率密度（补偿随机采样）

**为什么要除以PDF？**
- 某些方向采样概率高
- 但BRDF值可能小
- 除以PDF保证无偏

---

### 5. 俄罗斯轮盘赌

**问题**：
- 路径可能无限追踪下去
- 计算资源有限

**方法**：
- 每次反射后，随机决定是否继续
- 存活概率通常和反射率相关

**实现**：
```
存活概率 = min(1.0, 颜色亮度)
如果 随机数 > 存活概率：
    终止路径
否则：
    后续贡献 /= 存活概率  // 补偿
    继续追踪
```

**为什么有效？**
- 暗的路径大概率终止（省计算）
- 亮的路径大概率继续（重要）
- 除以存活概率保持无偏

## 直接光照优化

### 问题

纯路径追踪算直接光照效率低：
- 随机采样方向
- 只有很少方向能击中光源
- 大部分采样浪费

### 解决：直接采样光源

**方法**：
1. 在光源表面随机选一个点
2. 计算方向指向该点
3. 检查中间有没有遮挡
4. 计算贡献

**公式**：
```
直接光 = 光源亮度 × BRDF × cosθ × 几何项 / PDF光源
```

**PDF光源**：
- 在光源面积上的均匀采样
- 或者按亮度加权（重要光源多采）

**混合策略**：
- 既有BRDF采样（适合光滑表面）
- 又有光源采样（适合粗糙表面）
- 用多重重要性采样（MIS）结合

## 实际代码结构

```cpp
vec3 PathTrace(Ray ray, int depth) {
    // 找交点
    HitInfo hit = Intersect(ray, scene);
    if (!hit.hit) return BackgroundColor;
    
    // 击中光源
    if (hit.material.isEmitter) {
        return hit.material.emission;
    }
    
    // 俄罗斯轮盘赌
    float rrProb = min(1.0f, hit.material.reflectance.maxComponent());
    if (depth > 3 && Random() > rrProb) {
        return vec3(0);
    }
    
    // 采样BRDF
    vec3 wi;
    float pdf;
    vec3 brdf = hit.material.SampleBRDF(ray.direction, hit.normal, wi, pdf);
    
    // 生成新光线
    Ray newRay(hit.position, wi);
    
    // 递归追踪
    vec3 indirect = PathTrace(newRay, depth + 1);
    
    // 计算贡献
    vec3 color = brdf * indirect * dot(hit.normal, wi) / (pdf * rrProb);
    
    // 加上直接光照（可选，也可单独处理）
    color += SampleDirectLight(hit);
    
    return color;
}
```

## 本节梳理

| 步骤 | 做什么 | 关键点 |
|------|--------|--------|
| 发射光线 | 从相机穿过像素 | 可加抗锯齿、景深 |
| 找交点 | 光线和场景相交 | 用加速结构 |
| 检查光源 | 击中直接返回亮度 | 光源也是物体 |
| 俄罗斯轮盘赌 | 决定是否继续 | 保持无偏 |
| BRDF采样 | 选反射方向 | 根据材质类型 |
| 递归追踪 | 继续下一条光线 | 直到终止条件 |
| 计算贡献 | BRDF×cos×后续/PDF | 补偿采样概率 |

**关键理解**：
- 路径追踪 = 随机采样路径 + 正确计算贡献
- BRDF采样决定往哪走
- PDF补偿保证无偏
- 俄罗斯轮盘赌控制路径长度
