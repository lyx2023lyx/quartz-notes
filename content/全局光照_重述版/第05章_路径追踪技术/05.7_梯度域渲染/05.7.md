# 5.7 梯度域渲染

## 核心总结
梯度域渲染不直接算像素颜色，而是算像素间的颜色差异（梯度），然后通过求解方程重建图像，这样可以在相同采样下减少噪点。

---

## 为什么要用梯度？

### 直接渲染的问题

**方差大**：
- 像素颜色范围可能是0-10000
- 直接采样方差很大
- 噪点严重

**梯度更稳定**：
- 相邻像素颜色差异通常很小
- 梯度范围小（如-10到10）
- 方差小，容易收敛

### 打个比方

**直接渲染**：
- 问：北京今天气温多少？
- 答案：可能是0-30度之间，方差大

**梯度渲染**：
- 问：北京比天津高几度？
- 答案：大约2度，很确定

**重建**：
- 知道天津温度 + 温差 = 北京温度

---

## 梯度域的基本思想

### 图像梯度

**定义**：
梯度 = 相邻像素的差值

**两种梯度**：
- 水平梯度：右边像素 - 左边像素
- 垂直梯度：下面像素 - 上面像素

**数学表达**：
```
∇x(I) = I(x+1, y) - I(x, y)  // 水平
∇y(I) = I(x, y+1) - I(x, y)  // 垂直
```

### 梯度域渲染流程

```
1. 渲染梯度图（而不是颜色图）
2. 梯度图去噪（方差小，容易去噪）
3. 从梯度重建图像（求解泊松方程）
```

---

## 怎么渲染梯度？

### 差分采样

**水平梯度采样**：
- 在像素A发射一条光线
- 在像素B（右边相邻）发射一条光线
- 梯度 = 颜色B - 颜色A

**问题**：
- 每个梯度要采样两个像素
- 计算量翻倍

### 共享采样

**改进**：
- 相邻梯度共享像素采样
- 每个像素只采样一次
- 用于计算左右、上下多个梯度

**效率**：
- 和直接渲染差不多
- 但得到的是梯度信息

---

## 梯度去噪

### 为什么梯度更容易去噪？

**数值范围**：
- 颜色：0到无限（HDR）
- 梯度：通常-100到100

**方差**：
- 颜色方差大（不同材质亮度差异大）
- 梯度方差小（相邻像素通常相似）

### 去噪方法

**简单平均**：
```
梯度(x,y) = 平均(梯度(x±1,y), 梯度(x,y±1))
```

**边缘感知**：
- 检查颜色差异
- 差异大的地方不平均

**对应图片讲解**：
- 图片展示了梯度图和颜色图的对比
- 颜色图噪点严重
- 梯度图相对平滑

---

## 泊松重建

### 问题定义

**已知**：
- 水平梯度图 Gx
- 垂直梯度图 Gy

**求**：
- 图像 I

**约束**：
- I(x+1,y) - I(x,y) ≈ Gx(x,y)
- I(x,y+1) - I(x,y) ≈ Gy(x,y)

### 泊松方程

**目标**：
让重建图像的梯度尽量接近测量的梯度

**最小化**：
```
∫ |∇I - G|² dx dy
```

**求解**：
通过求解泊松方程：
```
∇²I = div(G)
```

其中：
- ∇²I：图像的拉普拉斯（二阶梯度）
- div(G)：梯度的散度

### 求解方法

**直接求解**：
- 构建大规模线性方程组
- 用稀疏矩阵求解器
- 慢但精确

**迭代求解**：
- 梯度下降
- 共轭梯度法
- 更快，适合GPU

**对应图片讲解**：
- 图片展示了泊松重建过程
- 输入带噪的梯度
- 输出平滑的图像
- 保留了边缘细节

---

## L1 vs L2 重建

### L2重建（标准泊松）

**目标**：
```
最小化：|∇I - G|²
```

**特点**：
- 平均意义下最优
- 对异常值敏感（outliers会拉偏结果）

### L1重建

**目标**：
```
最小化：|∇I - G|
```

**特点**：
- 更鲁棒，对异常值不敏感
- 保持尖锐边缘
- 计算更复杂

### 对比

| 特性 | L2 | L1 |
|------|-----|-----|
| 平滑度 | 更平滑 | 更锐利 |
| 异常值 | 敏感 | 鲁棒 |
| 计算 | 简单 | 复杂 |
| 适用 | 一般场景 | 有明显边缘的场景 |

---

## 实际应用

### 梯度域路径追踪（G-PT）

**流程**：
```
1. 标准路径追踪 + 差分采样得到梯度
2. 对梯度进行滤波/去噪
3. 泊松重建得到最终图像
```

**效果**：
- 相同采样数下噪点更少
- 或达到相同质量需要更少采样

**对应图片讲解**：
- 图片对比了标准PT和G-PT
- 相同采样数，G-PT噪点更少
- 梯度重建后边缘更清晰

### 与其他技术结合

**梯度域 + 降噪**：
- 梯度本身先降噪
- 重建后再整体降噪
- 双重保障

**梯度域 + 时间累积**：
- 梯度图时间累积
- 更稳定的梯度估计
- 重建质量更好

---

## 优缺点

### 优点

1. **方差更小**：梯度方差通常小于颜色
2. **边缘保持**：L1重建能保留锐利边缘
3. **采样效率高**：相同采样下质量更好

### 缺点

1. **计算量**：需要求解泊松方程
2. **内存占用**：存储梯度图需要更多内存
3. **复杂场景**：动态物体、遮挡变化处理困难

---

## 本节梳理

| 概念 | 含义 |
|------|------|
| 梯度 | 相邻像素的差值 |
| 梯度域 | 处理差值而不是绝对值 |
| 泊松重建 | 从梯度恢复图像 |
| L2重建 | 最小二乘，平滑 |
| L1重建 | 最小绝对值，锐利 |

**核心流程**：
```
渲染梯度 → 梯度去噪 → 泊松重建 → 最终图像
```

**关键理解**：
- 梯度比颜色更稳定
- 泊松方程把梯度变回图像
- 适合静态或慢变场景
- 计算成本是主要限制
