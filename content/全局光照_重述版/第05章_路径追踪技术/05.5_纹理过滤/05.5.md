# 5.5 纹理过滤

## 核心总结
纹理过滤就是处理纹理放大缩小时的走样问题，用滤波器平滑像素值，让纹理在不同距离和角度下都看起来自然。

---

## 为什么需要纹理过滤？

### 纹理走样问题

**缩小纹理时（Minification）**：
- 纹理比像素多
- 多个纹理像素对应一个屏幕像素
- 如果只取一个，会丢失信息，出现摩尔纹

**放大纹理时（Magnification）**：
- 纹理比像素少
- 一个纹理像素对应多个屏幕像素
- 如果不处理，会出现锯齿

**透视变形时**：
- 远处纹理被压缩
- 近处纹理被拉伸
- 各向异性（不同方向变形不同）

### 图片对比

**对应图片讲解**：
- 图片展示了不采用纹理过滤时的走样现象
- 远处的纹理出现严重的摩尔纹（一闪一闪的条纹）
- 近处的纹理边界锯齿明显

---

## 基本滤波方法

### 1. 最近邻（Nearest Neighbor）

**怎么做**：
- 直接取离采样点最近的纹理像素值
- 不做任何插值

**效果**：
- 放大时：方块状，边缘锯齿
- 缩小时：闪烁、走样严重

**优点**：
- 最快，只需要一次读取

**缺点**：
- 质量最差
- 基本不用于高质量渲染

---

### 2. 双线性插值（Bilinear Interpolation）

**怎么做**：
- 取采样点周围4个纹理像素
- 按距离加权平均

**公式**：
```
颜色 = (1-u)(1-v)C00 + u(1-v)C10 + uvC11 + (1-u)vC01
```
- u, v：采样点相对于左下角的偏移（0-1）
- C00, C10, C11, C01：四个角的颜色

**效果**：
- 放大时：平滑过渡，无锯齿
- 缩小时：还是会有走样（因为只取4个点）

**优点**：
- 质量好
- 计算量适中

**缺点**：
- 缩小时不够（信息丢失）
- 各向同性（不能处理各向异性）

**对应图片讲解**：
- 图片展示了双线性插值的示意图
- 在水平和垂直方向分别做线性插值
- 四个角的像素按距离权重混合

---

### 3. 双三次插值（Bicubic Interpolation）

**怎么做**：
- 取周围16个纹理像素（4×4）
- 用三次多项式插值

**效果**：
- 比双线性更平滑
- 保留更多高频细节

**公式**：
使用三次卷积核（如Catmull-Rom或B-spline）

**优点**：
- 质量最好（在简单方法中）
- 边缘更平滑

**缺点**：
- 计算量大（16次纹理读取）
- 实时渲染很少用

**对应图片讲解**：
- 图片展示了双三次插值使用的卷积核形状
- 比双线性的三角形形状更平滑
- 考虑了更远像素的影响

---

## Mipmap技术

### 核心思想

**问题**：
- 缩小时如果只取4个点，会丢失大量信息
- 远处物体可能对应几百个纹理像素

**解决**：
- 预先计算纹理的缩小版本
- 形成一个金字塔结构

**Mipmap链**：
```
Level 0: 原始纹理 512×512
Level 1: 256×256（平均4个像素）
Level 2: 128×128
...
Level 9: 1×1（整个纹理的平均颜色）
```

### 怎么选Level？

**屏幕像素到纹理像素的映射**：
- 计算一个屏幕像素对应多少纹理像素
- 例如：对应16×16个纹理像素 → 选Level 4（16=2⁴）

**插值Level**：
- 可能正好在两个Level之间
- 在相邻两个Level做双线性插值
- 然后按距离混合（Trilinear Filtering）

**公式**：
```
Level = log2(纹理像素覆盖范围)
```

---

## 三线性过滤（Trilinear Filtering）

**怎么做**：
- 在两个相邻的Mipmap Level分别做双线性插值
- 然后按距离混合这两个结果

**效果**：
- Level之间平滑过渡
- 不会出现切换Level时的跳变

**代价**：
- 两次纹理读取
- 两次双线性插值

**对应图片讲解**：
- 图片展示了Mipmap金字塔结构
- 不同Level对应不同距离
- 三线性过滤在Level之间插值

---

## 各向异性过滤（Anisotropic Filtering）

### 问题

**斜着看纹理时**：
- 一个屏幕像素对应纹理上的矩形区域
- 水平方向需要很多纹理像素
- 垂直方向只需要很少
- Mipmap假设是正方形，会模糊

**例子**：
- 看地板，远处地板纹理被严重压缩
- 如果只用一个Mipmap Level，水平方向过度模糊

### 各向异性过滤原理

**怎么做**：
- 沿着纹理倾斜方向采样
- 取多个样本平均

**具体实现**：
- 2x：沿一个方向取2个样本
- 4x：取4个样本
- 8x, 16x：更多样本

**效果**：
- 远处纹理更清晰
- 减少倾斜时的模糊

**对应图片讲解**：
- 图片对比了各向同性vs各向异性过滤
- 各向异性下远处地板纹理更清晰
- 道路或地板的纹理保持细节

---

## 纹理过滤在光线追踪中的应用

### 特殊问题

**光线追踪vs光栅化**：
- 光栅化：知道屏幕像素和纹理的精确映射
- 光线追踪：只有光线方向，不知道 footprint

**Footprint估计**：
需要估计光线在纹理上覆盖的区域大小

**方法**：

#### 1. Ray Cones
- 追踪光线锥的扩散
- 根据距离估计覆盖范围

#### 2. Ray Differentials
- 发射辅助光线估计梯度
- 根据光线变化率估计 footprint

#### 3. Texture LOD
- 根据光线长度估计
- 简单但不够精确

### 实现细节

```cpp
float CalculateTextureLOD(Ray ray, float t, float rayConeWidth) {
    // t: 光线传播距离
    // rayConeWidth: 光线锥宽度
    
    // 估计纹理覆盖范围
    float footprint = rayConeWidth * t;
    
    // 转换为Mipmap Level
    float lod = log2(footprint);
    
    return lod;
}
```

## EWA过滤（Elliptical Weighted Average）

### 问题

**极端情况**：
- 纹理被严重拉伸
-  footprint是椭圆形的
- 普通各向异性不够好

### EWA原理

**怎么做**：
- 在椭圆区域内采样
- 按距离加权平均

**公式**：
```
权重 = exp(-u²/σu² - v²/σv²)
```

**优点**：
- 处理极端各向异性
- 质量最高

**缺点**：
- 计算量大
- 实时渲染很少用

---

## 本节梳理

| 方法 | 原理 | 适用场景 |
|------|------|---------|
| 最近邻 | 取最近像素 | 追求速度，质量无所谓 |
| 双线性 | 4像素加权 | 放大时 |
| 双三次 | 16像素加权 | 高质量静态图 |
| Mipmap | 预计算金字塔 | 缩小和距离变化 |
| 三线性 | Level间插值 | 平滑过渡 |
| 各向异性 | 沿方向多采样 | 倾斜表面 |
| EWA | 椭圆区域采样 | 极端变形 |

**关键理解**：
- 走样是因为信息丢失
- 过滤是估计丢失的信息
- Mipmap+三线性是最常用的平衡
- 各向异性处理倾斜视角
