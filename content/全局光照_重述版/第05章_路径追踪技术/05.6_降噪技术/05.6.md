# 5.6 降噪技术

## 核心总结
路径追踪噪点太多，降噪技术用少量采样+智能平滑，在保证细节的同时消除噪点，让画面既快又好。

---

## 为什么路径追踪噪点多？

### 根本原因

**蒙特卡洛的本质**：
- 随机采样估算积分
- 采样少 → 方差大 → 噪点多
- 采样多 → 时间久

**噪点来源**：
1. **直接光照采样**：光源小、距离远时难击中
2. **间接光照采样**：路径长、贡献小
3. **镜面/光泽反射**：方向性强、难采样
4. **焦散效果**：光线路径特殊、采样效率低

### 噪点类型

**低频噪点**：
- 大块的颜色波动
- 采样分布不均匀
- 容易平滑

**高频噪点**：
- 细碎的颗粒感
- 方差太大
- 容易丢失细节

**对应图片讲解**：
- 图片展示了路径追踪的噪点效果
- 低采样时明显的颗粒感
- 高采样才逐渐平滑

---

## 降噪基本思路

### 核心思想

**用信号处理的角度看**：
- 噪点 = 高频噪声
- 图像 = 低频信号
- 降噪 = 低通滤波（保留低频，去除高频）

**但有个问题**：
- 边缘也是高频（颜色变化快）
- 直接模糊会丢失边缘

**解决**：
- 边缘感知滤波
- 只在相似颜色区域平均
- 保持边缘清晰

---

## 基础降噪方法

### 1. 高斯滤波（Gaussian Filter）

**原理**：
- 用高斯函数做权重
- 离中心越远，权重越小

**公式**：
```
权重 = exp(-(x²+y²)/(2σ²))
```

**问题**：
- 会模糊边缘
- 不适合直接用于降噪

**改进**：
- 双边滤波
- 引导滤波

---

### 2. 双边滤波（Bilateral Filter）

**核心思想**：
- 空间距离近 + 颜色相似的才平均
- 边缘两边颜色不同，不会互相影响

**两个权重**：
```
权重 = 空间权重 × 颜色权重

空间权重 = exp(-空间距离²/(2σs²))
颜色权重 = exp(-颜色差²/(2σr²))
```

**效果**：
- 平滑噪点
- 保持边缘

**对应图片讲解**：
- 图片对比了高斯滤波和双边滤波
- 高斯滤波边缘模糊
- 双边滤波边缘清晰

---

### 3. 引导滤波（Guided Filter）

**核心思想**：
- 用一个"引导图"决定怎么滤波
- 引导图可以是颜色、法线、深度等

**原理**：
- 假设输出是引导图的线性变换
- 局部保持边缘
- 计算效率高

**公式**：
```
输出 = a × 引导 + b
```
a, b 是局部窗口内算出来的

**优点**：
- 比双边滤波快
- 梯度保持好

---

## 高级降噪技术

### 1. SVGF（Spatiotemporal Variance-Guided Filtering）

**核心思想**：
- 空间滤波 + 时间滤波（多帧累积）
- 用方差指导滤波强度

**空间滤波**：
- 当前帧内滤波
- 用边缘停止函数（Edge-stopping Function）

**时间滤波**：
- 和前一帧混合
- 运动向量对齐像素

**方差指导**：
- 方差大的地方（噪点多）：多滤波
- 方差小的地方（干净）：少滤波

**对应图片讲解**：
- 图片展示了SVGF的效果
- 低采样输入
- 滤波后平滑且细节保留

---

### 2. BMFR（Blockwise Multi-Frame Regression）

**核心思想**：
- 分块回归
- 用历史帧信息预测当前帧

**做法**：
1. 把图像分成小块
2. 每块内做线性回归
3. 用特征（法线、深度、颜色）预测
4. 累积多帧信息

**优点**：
- 适合动态场景
- 计算效率高

---

### 3. 神经网络降噪（Intel OIDN, NVIDIA OptiX）

**核心思想**：
- 用深度学习学降噪
- 训练数据：低采样噪点图 + 高采样参考图

**网络结构**：
- U-Net结构
- 输入：噪点图 + 特征图（法线、深度、反照率）
- 输出：干净图

**训练**：
- 大量场景的数据对
- 损失函数：MSE或感知损失

**优点**：
- 质量最好
- 可以处理复杂噪点

**缺点**：
- 需要训练
- 对训练外的场景可能不好

**对应图片讲解**：
- 图片展示了神经网络降噪的效果
- 输入：1 sample/px（噪点严重）
- 输出：接近128 samples/px的质量

## 特征图（Feature Buffers）

### 什么是特征图？

**渲染时额外输出的信息**：
- 法线（Normal）
- 深度（Depth）
- 反照率（Albedo/Base Color）
- 材质ID
- 运动向量（Motion Vector）

### 为什么有用？

**法线**：
- 知道表面朝向
- 法线不同的地方不要混合

**深度**：
- 知道距离
- 深度差异大的不要混合

**反照率**：
- 知道基础颜色
- 降噪只针对光照部分

**运动向量**：
- 知道像素移动到哪里
- 时间滤波时对齐像素

### 使用方式

```cpp
// 判断两个像素是否可以混合
bool CanBlend(pixel1, pixel2) {
    if (abs(depth1 - depth2) > threshold) return false;
    if (dot(normal1, normal2) < threshold) return false;
    if (distance(albedo1, albedo2) > threshold) return false;
    return true;
}
```

## 时间累积（Temporal Accumulation）

### 核心思想

**用时间换质量**：
- 当前帧只有少量采样
- 但多帧累积起来采样就多了

**做法**：
1. 当前帧渲染（如1 sample/px）
2. 查找前一帧对应像素（用运动向量）
3. 混合当前帧和历史帧
4. 历史帧的贡献随时间衰减

**公式**：
```
当前输出 = 当前帧 × α + 历史帧 × (1-α)
```
α 是混合因子（如0.1）

### 问题：鬼影（Ghosting）

**现象**：
- 物体移动后， trailing effect
- 历史帧的信息残留在新位置

**解决**：
- 检测像素是否被遮挡
- 如果历史像素无效，重置历史

**对应图片讲解**：
- 图片展示了鬼影现象
- 历史帧混合过多导致拖影
- 改进后清晰无拖影

## 实际降噪流程

### 典型管线

```
1. 路径追踪渲染（1-4 samples/px）
   ↓
2. 特征图生成（法线、深度、反照率）
   ↓
3. 时间累积（和前一帧混合）
   ↓
4. 空间滤波（双边滤波或神经网络）
   ↓
5. 后处理（色调映射等）
```

### 参数调优

**滤波半径**：
- 太小：去不掉噪点
- 太大：丢失细节
- 自适应：根据局部方差调整

**混合因子**：
- 静态场景：可以累积很多帧
- 动态场景：减少历史权重

---

## 本节梳理

| 方法 | 原理 | 效果 |
|------|------|------|
| 高斯滤波 | 空间距离加权 | 会模糊边缘 |
| 双边滤波 | 空间+颜色加权 | 保持边缘 |
| 引导滤波 | 线性回归 | 快速、效果好 |
| SVGF | 时空方差指导 | 实时渲染标准 |
| BMFR | 分块回归 | 适合动态场景 |
| 神经网络 | 深度学习 | 质量最高 |

**关键理解**：
- 降噪 = 平滑噪点 + 保持细节
- 特征图帮助判断边缘
- 时间累积用历史帧信息
- 现代游戏用SVGF或神经网络
