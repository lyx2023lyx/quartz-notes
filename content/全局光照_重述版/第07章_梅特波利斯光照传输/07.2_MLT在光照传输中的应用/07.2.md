# 7.2 MLT在光照传输中的应用

## 核心总结
将梅特波利斯算法应用到光照传输中，通过在路径空间随机游走，重点关注贡献大的路径，解决传统方法难以采样的困难光照场景。

---

## 路径空间

### 什么是路径空间？

**定义**：
所有可能的光线路径的集合

**路径表示**：
```
P = (x₀, x₁, x₂, ..., xₖ)
```
- x₀：相机/眼睛位置
- x₁, x₂, ...：光线经过的表面点
- xₖ：光源位置

**维度**：
- 每个点3个坐标
- k个点就是3k维空间
- 非常高维！

### 路径贡献函数

**定义**：
```
f(P) = 路径P对图像的贡献
```

**计算**：
```
f(P) = 几何项 × BRDF乘积 × 光源亮度
```

**目标**：
以概率正比于f(P)的方式采样路径

---

## MLT渲染流程

### 完整算法

```
1. 初始化：
   - 用双向路径追踪生成初始路径
   - 计算初始贡献

2. 预热（Burn-in）：
   - 运行N次迭代，不记录结果
   - 让马尔可夫链达到稳态

3. 主循环：
   对于每次迭代：
      a. 选择突变类型（随机或轮询）
      b. 应用突变得到新路径
      c. 计算新路径贡献
      d. 计算接受概率
      e. 决定是否接受
      f. 如果接受：当前路径 = 新路径
      g. 记录当前路径到图像

4. 输出：
   - 累积的图像除以样本数
```

### 图像累积

**每个路径记录到哪里**：
- 路径从相机出发，穿过某个像素
- 该路径的贡献加到对应像素

**注意**：
- 一个路径只影响一个像素
- 多次迭代后每个像素有很多样本

## 与其他方法结合

### MLT + 双向路径追踪

**初始化**：
- 用双向路径追踪生成好的初始路径
- 提高MLT启动质量

**混合**：
- 大部分样本用MLT
- 偶尔用双向路径追踪探索新区域

### MLT + 光子映射

**互补**：
- MLT：擅长困难可见性
- 光子映射：擅长焦散

**结合**：
- 用VCM（顶点连接与合并）框架
- 统一处理各种路径

## 实际应用

### 困难场景案例

#### 小窗户室内

**场景**：
- 大房间，只有一个小窗户
- 大部分光照来自间接反射

**问题**：
- 路径追踪：很难采样到从窗户进来的光
- 噪点集中在阴影区域

**MLT解决**：
- 一旦找到一条从窗户进来的路径
- 在该路径附近大量采样
- 有效填充暗部细节

#### 镜子迷宫

**场景**：
- 很多镜子相互反射
- 复杂的光线路径

**问题**：
- 路径追踪：随机方向很难正确反射
- 需要非常长的路径

**MLT解决**：
- 好的长路径被保留和修改
- 逐步优化反射链

### 实际渲染器

**LuxRender**：
- 开源物理渲染器
- 支持MLT

**Mitsuba**：
- 研究用渲染器
- 完整的MLT实现

**商业软件**：
- 某些高端离线渲染器使用MLT变体
- 主要用于电影、广告

## 调参技巧

### 突变大小

**太小**：
- 探索太慢
-  stuck在局部

**太大**：
- 接受率太低
- 退化成独立采样

**经验**：
- 接受率控制在20-40%
- 自适应调整

### 预热时间

**太短**：
- 还没达到稳态
- 结果有偏

**太长**：
- 浪费时间

**经验**：
- 通常是总样本数的10-20%
- 观察贡献值是否稳定

### 多种突变平衡

**小突变**：
- 精细化好路径
- 占70%

**大突变**：
- 探索新区域
- 占30%
