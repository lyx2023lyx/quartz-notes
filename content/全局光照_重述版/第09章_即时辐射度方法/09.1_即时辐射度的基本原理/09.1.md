# 9.1 即时辐射度的基本原理

## 核心总结
即时辐射度用虚拟点光源(VPL)近似间接光照，从光源发射光子，在漫反射面存储为VPL，渲染时直接计算VPL的贡献。

---

## 为什么叫"即时"辐射度？

### 传统辐射度的问题

**预计算**：
- 形状系数需要预计算
- 场景必须静态
- 改变几何就要重新算

### 即时辐射度的解决

**思想**：
- 不预计算形状系数
- 实时生成虚拟光源
- 每帧重新计算

**代价**：
- 计算量大
- 适合离线或高质量实时

---

## 核心概念：虚拟点光源(VPL)

### 什么是VPL？

**定义**：
- 从光源发出的光子
- 击中漫反射表面后
- 把该点当成一个新光源

**特点**：
- 不是真的光源
- 只是近似间接光
- 位置在漫反射面上

### VPL怎么产生？

**步骤**：
1. 从真实光源发射光子
2. 追踪光子路径
3. 每次击中漫反射面，创建一个VPL
4. 继续追踪，直到终止

**例子**：
```
光源 → 镜面反射 → 墙面 → 创建VPL
```

---

## 渲染过程

### 第一步：生成VPL

```
对于每个光子：
    从光源发射
    追踪路径
    遇到漫反射面：
        存储VPL（位置、方向、能量）
    继续追踪（俄罗斯轮盘赌）
```

### 第二步：用VPL渲染

```
对于每个像素：
    发射相机光线
    找交点
    计算光照：
        直接光照（真实光源）
        间接光照（所有VPL的贡献和）
```

### VPL贡献计算

```
亮度 = VPL能量 × BRDF × 几何项 / 距离²
```

---

## 优点和缺点

### 优点

1. **无需预计算**
   - 每帧重新生成VPL
   - 支持动态场景

2. **焦散效果好**
   - 光经过镜面后的VPL
   - 自然产生焦散

3. **实现相对简单**
   - 不需要解方程
   - 光线追踪即可

### 缺点

1. **VPL数量问题**
   - 太少：噪点
   - 太多：计算慢

2. **连接问题（SINGULARITY）**
   - 表面离VPL太近
   - 1/r² → 无穷大
   - 需要clamp

3. **阴影问题**
   - 每个VPL都要算阴影
   - 计算量大

---

## 本节梳理

**即时辐射度 = VPL近似间接光**

**两步走**：
1. 生成VPL（从光源发射光子）
2. 用VPL渲染（每个VPL贡献求和）

**关键问题**：
- VPL数量选择
- 近处连接singularities
- 阴影计算量大
