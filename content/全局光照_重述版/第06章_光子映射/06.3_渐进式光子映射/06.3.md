# 6.3 渐进式光子映射

## 核心总结
渐进式光子映射通过迭代发射更多光子并逐步减小查询半径，在保持无偏的同时提高效率，最后收敛到正确结果。

---

## 基本光子映射的问题

### 参数选择困难

**光子数 vs 查询半径**：
- 光子少：噪点多
- 光子的多：内存大
- k值小：噪点多
- k值大：太模糊

**一次性计算**：
- 光子图建好后不再改
- 结果质量固定
- 不能渐进改善

---

## 渐进式思想

### 核心想法

**像路径追踪一样迭代**：
- 第1轮：发射一些光子，算一个结果
- 第2轮：再发射一些，累积改进
- 第n轮：继续累积，越来越好

**关键调整**：
- 每轮减小查询半径
- 保证最终收敛

### 为什么能收敛

**直观理解**：
- 光子越来越多
- 查询半径越来越小
- 局部密度越来越准
- 偏差越来越小

**数学保证**：
- 如果半径 r ∝ 1/√N
- N→∞ 时，r→0
- 估计变成精确值

---

## 算法流程

### 初始化

```
总光子数 = 0
累积图像 = 黑色
查询半径 = 初始值（如整个场景大小）
```

### 迭代过程

```
对于每轮迭代 i：
    1. 发射M个新光子
    2. 累积到光子图中
    3. 更新总光子数 N += M
    4. 计算新半径 r = r0 / √N
    5. 渲染整个图像
    6. 累积到最终结果
```

### 半径收缩

**公式**：
```
r_i = r_0 / √N_i
```

**r_0是什么**：
- 初始半径
- 通常是场景包围盒大小
- 或者根据经验设定

**为什么除以√N**：
- 面积和半径平方成正比
- 光子密度和光子数成正比
- 要保持k个光子，半径需要这样收缩

---

## 累积与平均

### 简单平均的问题

**早期轮次**：
- 光子少，噪点多
- 不应该和后期轮次同等权重

### 加权累积

**权重设计**：
- 后期轮次权重更高
- 或者根据方差调整权重

**典型方案**：
```
最终结果 = Σ(第i轮结果 × w_i) / Σw_i
```

**w_i选什么**：
- 可以都等于1（简单平均）
- 可以和光子数成正比
- 或者根据质量自适应

## 与基本光子映射对比

| 特性 | 基本光子映射 | 渐进式光子映射 |
|------|-------------|---------------|
| 光子数 | 固定 | 逐渐增加 |
| 查询半径 | 固定 | 逐渐减小 |
| 内存 | 固定 | 持续增长 |
| 结果 | 一次性 | 渐进改善 |
| 偏差 | 固定 | 逐渐减小 |
| 适用 | 快速预览 | 高质量渲染 |

---

## 实际应用

### 什么时候用

**高质量离线渲染**：
- 电影、动画
- 产品可视化
- 建筑可视化

**需要焦散效果**：
- 渐进式特别适合焦散
- 可以精确控制质量

### 实现技巧

**光子复用**：
- 每轮只发射新光子
- 旧光子不删除
- 加速结构动态更新

**并行渲染**：
- 每轮可以并行
- 多线程发射光子
- GPU加速渲染

**自适应终止**：
- 达到目标质量就停
- 不用固定轮数
- 省时间

---

## 本节梳理

**核心思想**：
```
迭代发射光子 + 缩小半径 = 渐进收敛
```

**关键公式**：
```
r = r_0 / √N
```

**优势**：
- 无偏（最终）
- 可控质量
- 渐进改善

**代价**：
- 内存持续增长
- 计算时间更长
- 实现更复杂
