# 6.2 基本光子映射技术

## 核心总结
光子映射分两步：第一步从光源发射光子并存储在光子图中，第二步从相机发射光线并用光子密度估计计算漫反射光照。

---

## 双通道方法

### 第一步：光子追踪（Photon Tracing）

**从光源出发**：
- 发射大量携带能量的光子
- 光子在场景中反射/折射
- 在漫反射表面存储光子

**存储什么**：
- 光子位置（世界坐标）
- 光子方向（入射方向）
- 光子能量（辐射通量）
- 可能还有波长/RGB

### 第二步：渲染通道（Rendering Pass）

**从相机出发**：
- 发射光线（路径追踪）
- 击中表面时计算光照
- 漫反射部分查光子图
- 镜面部分继续追踪

---

## 光子发射

### 光源采样

**均匀发射**：
- 点光源：均匀分布在整个球面
- 面积光源：在面积上均匀采样
- 聚光灯：按强度分布采样

**能量分配**：
```
单个光子能量 = 光源总能量 / 光子数量
```

**重要性采样**：
- 亮的光源多发光子
- 暗的光源少发光子
- 或者直接按亮度比例分配

### 光子追踪过程

```
对于每个光子：
    从光源发射
    循环直到终止：
        找最近交点
        
        如果表面是漫反射：
            存储光子到光子图
            俄罗斯轮盘赌决定是否继续
        
        如果表面是镜面/光泽：
            按反射/折射方向继续
            能量按BRDF衰减
        
        如果击中光源或天空：
            终止（吸收）
```

**对应图片讲解**：
- 图片展示了不同光源发射光子的分布
- 点光源：球面均匀分布
- 聚光灯：锥形分布
- 面积光源：平面分布

## 光子图结构

### 简单版本：数组

```cpp
struct Photon {
    vec3 position;  // 位置
    vec3 direction; // 入射方向
    vec3 power;     // 能量
};

vector<Photon> photonMap;
```

### 高效查询：KD树

**为什么用KD树**：
- 需要快速找最近邻
- k-NN查询是核心操作

**构建**：
- 按位置坐标递归分割
- 每个节点存储一个光子
- 叶子节点包含多个光子

**查询**：
- 从根节点开始
- 根据位置决定走左还是右
- 维护一个最近k个光子的优先队列

## 辐射亮度估计

### 基本公式

**从光子密度到辐射亮度**：
```
L(x, ω) ≈ (1/πr²) × Σ K(Δx/r) × φ × f_r(ω, ω_i)
```

**各项含义**：
- x：查询点位置
- ω：出射方向（看向相机的方向）
- r：查询半径（包含k个光子的球半径）
- K：核函数
- φ：光子能量
- f_r：BRDF
- ω_i：光子入射方向

### 简化版本（平顶核）

**如果核函数是平顶**（半径内权重为1，外为0）：
```
L(x, ω) = (1/πr²) × Σ φ × f_r(ω, ω_i)
```

**步骤**：
1. 找距离x最近的k个光子
2. 计算第k个光子的距离r
3. 对这k个光子求和：光子能量 × BRDF
4. 除以 πr²

### BRDF的作用

**为什么乘BRDF**：
- 光子携带的是入射光
- 我们要算出射光
- BRDF告诉入射光怎么反射

**漫反射简化**：
- 如果是理想漫反射
- BRDF = 反照率 / π
- 可以提到求和外面

## 渲染时的光照计算

### 完整流程

```
对于每个像素：
    发射相机光线
    
    循环直到终止：
        找交点
        
        直接光照：
            采样光源
            计算阴影
        
        间接光照（漫反射）：
            光子密度估计
            
        镜面/光泽反射：
            继续追踪
            或查环境贴图
```

### 分离BRDF

**通常分成两部分**：
1. **镜面/光泽部分**：用路径追踪继续采样
2. **漫反射部分**：用光子图估计

**为什么这样分**：
- 镜面反射方向性强
- 光子图对镜面没用（方向不匹配）
- 漫反射方向均匀，适合光子图

## 光子映射的优缺点

### 优点

1. **焦散效果好**
   - 专门擅长处理光线聚焦效果
   - 游泳池底部的光斑
   - 玻璃杯的亮纹

2. **多次漫反射**
   - 光子可以反弹很多次
   - 每次都在漫反射面存储
   - 适合处理室内间接光照

3. **噪点少**
   - 密度估计平滑结果
   - 比纯路径追踪干净

4. **一致性**
   - 光子数→∞时正确
   - 有偏但可用

### 缺点

1. **内存占用大**
   - 需要存储大量光子
   - 几十万个到几百万个

2. **有偏估计**
   - 会模糊细节
   - 边缘可能不清晰

3. **参数调优**
   - k值选多少？
   - 光子数多少？
   - 需要经验

4. **动态场景难**
   - 光子图需要重建
   - 不适合实时渲染

---

## 本节梳理

**双通道流程**：
```
光子追踪（光源→场景）
    ↓
存储光子图
    ↓
渲染通道（相机→场景）
    ↓
光子密度估计计算漫反射
```

**核心公式**：
```
辐射亮度 = (1/πr²) × Σ(光子能量 × BRDF)
```

**适用场景**：
- 焦散效果
- 多次漫反射
- 室内场景
- 离线渲染
