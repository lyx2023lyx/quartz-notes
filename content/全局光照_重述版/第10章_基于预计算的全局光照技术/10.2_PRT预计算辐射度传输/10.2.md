# 10.2 PRT（预计算辐射度传输）

## 核心总结
PRT预计算光线在场景中的传输方式，用球谐函数表示，实时时只需环境光照系数和预计算的传输系数点积即可得到光照。

---

## PRT的核心思想

### 离线预计算

**计算什么**：
物体表面每个点，从各个方向来的光，经过场景传输后，有多少能到达该点。

**结果**：
传输函数 T(位置, 入射方向)

**存储**：
用球谐函数表示 T
每个点存一组球谐系数

### 实时计算

**环境光照**：
也投影到球谐函数
得到系数 L

**最终光照**：
```
光照 = Σ T_coefficients × L_coefficients
```
就是点积！

---

## 为什么能分离？

### 渲染方程简化

**假设**：
1. 场景静态（几何不变）
2. 材质漫反射
3. 光源可以变化

**方程**：
```
出射光 = 传输函数 × 入射光
```

**卷积形式**：
```
L_out(ω) = ∫ T(ω, ω') × L_in(ω') dω'
```

**球谐域**：
```
c_out = Σ t_l × l_l
```
变成逐系数相乘！

---

## 预计算内容

### 对于每个顶点

**需要预计算**：
- 可见性（被遮挡的方向）
- 多次反射贡献
- 相互反射

**存储**：
- 3阶球谐：9个RGB系数 = 27个浮点数
- 4阶球谐：16个RGB系数 = 48个浮点数

### 内存占用

**顶点数 × 系数数 × 3(RGB)**

**例子**：
- 1000个顶点
- 3阶球谐
- 1000 × 9 × 3 = 27KB
- 很小！

---

## 实时渲染流程

### 离线阶段

```
1. 构建场景
2. 对每个顶点：
   - 计算传输函数
   - 投影到球谐
   - 存储系数
3. 导出数据
```

### 实时阶段

```
1. 获取环境光照
2. 投影到球谐（得到L系数）
3. 对每个顶点：
   - 点积：T系数 · L系数
   - 得到光照
4. 渲染
```

---

## 优缺点

### 优点

1. **实时性好**
   - 计算只是点积
   - 可以在GPU上并行

2. **质量高**
   - 可以预计算多次反射
   - 软阴影、间接光都有

3. **光源动态**
   - 环境光可以实时变化
   - 时间变化、天气变化

### 缺点

1. **场景必须静态**
   - 几何不能变
   - 材质不能变（严格来说）

2. **只处理漫反射**
   - 镜面反射需要其他方法

3. **预计算时间长**
   - 复杂场景要算很久
   - 每次修改要重新算

## 本节梳理

**PRT = 预计算传输 + 实时点积**

**公式**：
```
光照 = T_coefficients · L_coefficients
```

**适用**：
- 静态场景
- 动态环境光
- 漫反射材质

**优势**：
- 实时
- 高质量
- 简单
