# 骨骼动画基础

## 什么是骨骼动画

**骨骼动画（Skeletal Animation）**，也称为**蒙皮动画（Skinned Animation）**，是计算机图形学中用于角色动画的一种技术。它将角色的几何体（网格）与一套骨骼结构关联，通过驱动骨骼的运动来带动角色模型的变形。

### 核心思想

```
传统顶点动画:          骨骼动画:
┌─────────────┐        ┌─────────────┐
│  顶点1位置   │        │    骨骼     │
│  顶点2位置   │   VS   │  ┌──┬──┐   │
│  顶点3位置   │        │  │  │  │   │
│    ...      │        │  └──┴──┘   │
│  (每帧存储   │        │    │      │
│   所有顶点)  │        │   绑定     │
└─────────────┘        │    │      │
                       │  ┌──┴──┐   │
                       │  │ 网格  │   │
                       │  └─────┘   │
                       └─────────────┘
```

**骨骼动画的优势：**
- **数据量小**：存储骨骼变换而非每个顶点位置
- **可复用**：同一套骨骼可驱动不同角色模型
- **易于混合**：多动画混合、过渡更加自然
- **程序化控制**：可通过 IK 等技术实时修改

## 骨骼动画的核心概念

### 1. 骨架（Skeleton）

骨架是由一系列**关节（Joint/Bone）**组成的层级结构，类似人体的骨骼系统。

```
骨架层级结构示例:

                    Root (根关节)
                      │
          ┌───────────┼───────────┐
          │           │           │
       Spine       L_Thigh    R_Thigh
          │           │           │
       Chest       L_Calf     R_Calf
          │           │           │
     ┌────┴────┐   L_Foot     R_Foot
     │         │
 L_Arm    R_Arm
     │         │
L_Forearm R_Forearm
     │         │
 L_Hand   R_Hand
```

**关节的关键属性：**
- **名字（Name）**：唯一标识
- **父索引（Parent Index）**：层级关系
- **变换（Transform）**：平移、旋转、缩放
- **绑定姿势（Bind Pose / Rest Pose）**：默认T姿势或A姿势

---

### 2. 绑定姿势（Bind Pose / Rest Pose）

绑定姿势是角色在**未动画状态**下的默认姿态，通常采用 **T-Pose** 或 **A-Pose**。

```
T-Pose:                  A-Pose:
    │                      \\   //
   ─┼─                     \\ //
    │                       \//
   / \\                      /\\
  /   \\                    /  \\
```

**绑定姿势的作用：**
- 作为动画的参考基准
- 计算蒙皮权重的基础
- 确定关节的初始变换

---

### 3. 动画数据（Animation）

动画数据描述了**每个关节随时间变化的变换**。

```
动画数据结构:

Animation
├── 时长 (Duration)
├── 采样率 (Sample Rate)
└── 关节轨道[] (Joint Tracks)
    ├── Joint 0 Track
    │   ├── Keyframe 0: time=0.0s,  transform=...
    │   ├── Keyframe 1: time=0.1s,  transform=...
    │   └── Keyframe 2: time=0.2s,  transform=...
    ├── Joint 1 Track
    │   └── ...
    └── ...
```

**关键帧（Keyframe）**：在特定时间点记录关节的变换。

**采样（Sampling）**：在两个关键帧之间进行插值计算，得到当前时间的变换。

---

### 4. 蒙皮（Skinning）

蒙皮是将**网格顶点绑定到骨骼**的过程。每个顶点可以受一个或多个关节影响，通过**权重**决定影响程度。

```
蒙皮权重示例:

顶点A ────[0.6]────▶ 肩关节
     \
      ────[0.3]────▶ 肘关节
       \
        ───[0.1]────▶ 腕关节

顶点B ────[0.8]────▶ 肘关节
     \
      ────[0.2]────▶ 腕关节
```

**蒙皮算法（顶点变换）：**

```
顶点最终位置 = Σ(权重i × 关节i的变换矩阵 × 顶点初始位置)

数学表达:
    P_final = Σ(w_i × M_i × P_bind)
    
其中:
    w_i = 第i个关节的权重
    M_i = 第i个关节的模型空间变换矩阵
    P_bind = 顶点在绑定姿势下的位置
```

---

### 5. 坐标空间

骨骼动画涉及多个坐标空间，理解它们至关重要：

```
坐标空间层级:

世界空间 (World Space)
    │
    ├── 角色A的模型空间 (Model Space)
    │       │
    │       ├── 根关节空间
    │       │       ├── 子关节1空间
    │       │       │       └── ...
    │       │       └── 子关节2空间
    │       │
    │       └── 绑定姿势空间 (Bind Space)
    │               └── 顶点初始位置
    │
    └── 角色B的模型空间
            └── ...
```

| 空间 | 说明 |
|------|------|
| **局部空间（Local Space）** | 相对于父关节的坐标空间 |
| **模型空间（Model Space）** | 相对于角色原点的坐标空间 |
| **世界空间（World Space）** | 相对于场景原点的坐标空间 |
| **绑定空间（Bind Space）** | 绑定姿势下的坐标空间 |

**关键矩阵：**
- **Local Matrix**：局部变换矩阵（相对于父关节）
- **Model Matrix**：模型变换矩阵（相对于角色原点）
- **Bind Pose Matrix**：绑定姿势矩阵
- **Inverse Bind Pose Matrix**：绑定姿势的逆矩阵

## 关键算法详解

### 1. 动画采样（Animation Sampling）

采样是在两个关键帧之间进行**插值**，得到当前时间的变换。

```cpp
// 线性插值示例
float Interpolate(float t, float key1, float key2, float time1, float time2) {
    float ratio = (t - time1) / (time2 - time1);
    return key1 + ratio * (key2 - key1);
}

// 四元数球面插值 (SLERP) - 用于旋转
Quaternion Slerp(Quaternion q1, Quaternion q2, float t) {
    float dot = Dot(q1, q2);
    
    // 处理负数dot，选择最短路径
    if (dot < 0) {
        q2 = -q2;
        dot = -dot;
    }
    
    // 如果角度很小，使用线性插值
    if (dot > 0.9995f) {
        return Normalize(q1 + t * (q2 - q1));
    }
    
    float theta0 = acos(dot);
    float theta = theta0 * t;
    float sinTheta = sin(theta);
    float sinTheta0 = sin(theta0);
    
    float s0 = cos(theta) - dot * sinTheta / sinTheta0;
    float s1 = sinTheta / sinTheta0;
    
    return s0 * q1 + s1 * q2;
}
```

### 2. 局部空间到模型空间（Local to Model）

将关节的局部变换转换为模型空间变换，需要**自顶向下**遍历骨骼层级。

```cpp
void ComputeModelMatrices(
    const Skeleton& skeleton,
    const Transform localTransforms[],  // 采样得到的局部变换
    Matrix4 modelMatrices[]             // 输出: 模型空间矩阵
) {
    // 根关节的模型矩阵 = 局部矩阵
    modelMatrices[0] = localTransforms[0].ToMatrix();
    
    // 遍历其他关节 (深度优先)
    for (int i = 1; i < skeleton.num_joints(); ++i) {
        int parent = skeleton.parent(i);
        
        // 模型矩阵 = 父关节模型矩阵 × 局部矩阵
        modelMatrices[i] = modelMatrices[parent] * localTransforms[i].ToMatrix();
    }
}
```

---

### 3. 蒙皮矩阵计算（Skinning Matrix）

蒙皮矩阵将顶点从绑定姿势变换到当前姿势。

```cpp
void ComputeSkinningMatrices(
    const Skeleton& skeleton,
    const Matrix4 modelMatrices[],       // 当前模型空间矩阵
    Matrix4 skinningMatrices[]           // 输出: 蒙皮矩阵
) {
    for (int i = 0; i < skeleton.num_joints(); ++i) {
        // 蒙皮矩阵 = 当前模型矩阵 × 绑定姿势逆矩阵
        skinningMatrices[i] = modelMatrices[i] * skeleton.inverse_bind_pose(i);
    }
}
```

## 动画混合（Blending）

动画混合是将多个动画按权重组合，实现平滑过渡或层叠效果。

```
动画混合示例:

行走动画 (weight=0.7) ──┐
                        ├─▶ 混合结果
跑步动画 (weight=0.3) ──┘

或:

基础姿势 (weight=1.0) ───┐
                         ├─▶ 基础 + 受伤动画叠加
受伤动画 (weight=0.5) ───┘
```

### 混合公式

```cpp
// 变换混合
Transform BlendTransforms(
    const Transform transforms[],
    const float weights[],
    int numAnimations
) {
    Transform result;
    
    // 混合平移
    result.translation = Vector3(0, 0, 0);
    for (int i = 0; i < numAnimations; ++i) {
        result.translation += weights[i] * transforms[i].translation;
    }
    
    // 混合旋转 (使用四元数SLERP的近似)
    result.rotation = transforms[0].rotation;
    for (int i = 1; i < numAnimations; ++i) {
        result.rotation = Slerp(result.rotation, transforms[i].rotation, weights[i]);
    }
    
    // 混合缩放
    result.scale = Vector3(1, 1, 1);
    for (int i = 0; i < numAnimations; ++i) {
        result.scale += weights[i] * (transforms[i].scale - Vector3(1,1,1));
    }
    
    return result;
}
```

### 混合类型

| 类型 | 说明 | 用途 |
|------|------|------|
| **线性混合（LERP）** | 两个动画之间线性插值 | 动画过渡 |
| **加法混合（Additive）** | 将增量动画叠加到基础动画 | 细节层叠 |
| **部分混合（Partial）** | 只对部分关节进行混合 | 上下身分离 |

## 动画压缩

为了减少内存占用和加载时间，动画数据通常需要压缩。

### 常见压缩技术

```
压缩技术对比:

┌─────────────────┬─────────────────┬─────────────────┐
│    技术         │     压缩率      │     质量损失    │
├─────────────────┼─────────────────┼─────────────────┤
│ 原始数据        │     1x          │     无          │
│ 关键帧减少      │     2-5x        │     低          │
│ 四元数量化      │     2x          │     极低        │

```

### 四元数量化

```cpp
// 将四元数量化为16位整数 (每分量)
struct QuaternionQuantized {
    uint16_t x, y, z;  // w 可通过计算得到
};

// 量化: float [-1, 1] -> uint16 [0, 65535]
uint16_t Quantize(float value) {
    return (uint16_t)((value + 1.0f) * 0.5f * 65535.0f);
}

// 反量化
float Dequantize(uint16_t value) {
    return (value / 65535.0f) * 2.0f - 1.0f;
}

// 恢复第4个分量
float w = sqrt(1.0f - x*x - y*y - z*z);
```

## 反向运动学（IK）

IK 是根据**末端位置**反推关节角度的技术。

```
正向运动学 (FK):        反向运动学 (IK):

    θ1                    目标位置
    │                       │
    ├───θ2───手部           │
                 求解 ──────┘
                 θ1, θ2, θ3
```

### 两骨骼 IK

最常用的 IK 类型，用于手臂和腿部。

```cpp
// 两骨骼 IK 伪代码
bool SolveTwoBoneIK(
    Vector3 start,           // 起点 (肩膀/臀部)
    Vector3 end,             // 终点 (手/脚)
    Vector3 target,          // 目标位置
    Vector3 bendHint,        // 弯曲方向提示
    float upperLength,       // 上臂/大腿长度
    float lowerLength,       // 前臂/小腿长度
    Quaternion& upperRot,    // 输出: 上关节旋转
    Quaternion& lowerRot     // 输出: 下关节旋转
) {
    // 1. 计算目标距离
    float targetDist = Distance(start, target);
    float maxReach = upperLength + lowerLength;
    
    // 2. 处理不可达情况
    if (targetDist > maxReach) {
        target = start + Normalize(target - start) * maxReach;
        targetDist = maxReach;
    }
    
    // 3. 使用余弦定理计算角度
    float upperAngle = acos((upperLength*upperLength + targetDist*targetDist - lowerLength*lowerLength) 
                            / (2 * upperLength * targetDist));
    
    // 4. 构建旋转...
    // ...
    
    return true;
}
```

## 总结

骨骼动画是现代游戏和实时图形中角色动画的核心技术。理解以下概念至关重要：

1. **骨架层级**：关节的父子关系
2. **绑定姿势**：动画的参考基准
3. **动画数据**：随时间变化的关节变换
4. **蒙皮**：顶点与骨骼的关联
5. **采样**：关键帧插值
6. **混合**：多动画组合
7. **IK**：程序化姿势修正
