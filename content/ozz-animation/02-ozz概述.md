# ozz-animation 概述

## 简介

**ozz-animation** 是一个开源的 **C++ 骨骼动画库**，专注于提供高性能、低内存占用的运行时角色动画功能。

- **官网/文档**: http://guillaumeblanc.github.io/ozz-animation/
- **GitHub**: https://github.com/guillaumeblanc/ozz-animation
- **许可证**: MIT License
- **作者**: Guillaume Blanc

### 核心定位

```

├─────────────────────────────────────────────────────────────────┤

│   │  (Maya等)   │      │  (fbx2ozz等)    │      │ (ozz动画)   │ │

│   • 高性能 & 低内存                                               │
└─────────────────────────────────────────────────────────────────┘
```

## 主要特性

### 1. 运行时特性

| 特性 | 说明 |
|------|------|
| **动画采样** | 高效的关键帧采样和插值 |
| **动画混合** | 多动画线性混合和加法混合 |
| **部分混合** | 关节级权重控制 |
| **反向运动学 (IK)** | 两骨骼IK和瞄准IK |
| **根运动** | 根骨骼运动提取和应用 |
| **用户通道** | 浮点/向量/四元数轨道动画 |
| **多线程支持** | 并行采样和蒙皮 |
| **SIMD优化** | SSE/AVX/FMA 加速 |

### 2. 离线工具链

| 工具 | 功能 |
|------|------|
| **fbx2ozz** | FBX 格式导入 (支持 DAE/OBJ/3DS/DXF) |
| **gltf2ozz** | glTF 格式导入 |
| **动画优化** | 自动关键帧减少和压缩 |
| **加法动画构建** | 生成增量动画 |
| **根运动提取** | 提取根骨骼运动轨迹 |

### 3. 平台支持

- **操作系统**: Linux, macOS, Windows
- **架构**: x86, x86-64, ARM, WebAssembly
- **编译器**: GCC, Clang, MSVC (C++17)

## 模块详解

### 1. ozz_base (基础库)

```cpp
// 核心功能
#include <ozz/base/platform.h>       // 平台抽象
#include <ozz/base/maths/vec_float.h> // 向量数学
#include <ozz/base/maths/quaternion.h> // 四元数
#include <ozz/base/maths/transform.h>  // 变换
#include <ozz/base/memory/allocator.h> // 内存分配器
#include <ozz/base/io/archive.h>       // 序列化
```

**特点**:
- 独立的数学库 (SoA SIMD)
- 可重定向的内存分配器
- 非侵入式序列化机制

---

### 2. ozz_animation (动画库)

#### 运行时 (Runtime)

```cpp
#include <ozz/animation/runtime/skeleton.h>
#include <ozz/animation/runtime/animation.h>
#include <ozz/animation/runtime/sampling_job.h>
#include <ozz/animation/runtime/blending_job.h>
#include <ozz/animation/runtime/local_to_model_job.h>
```

**核心类**:
- `Skeleton`: 运行时骨架
- `Animation`: 压缩后的动画数据
- `SamplingJob`: 动画采样任务
- `BlendingJob`: 动画混合任务
- `LocalToModelJob`: 局部空间到模型空间转换

#### 离线工具 (Offline)

```cpp
#include <ozz/animation/offline/raw_skeleton.h>
#include <ozz/animation/offline/raw_animation.h>
#include <ozz/animation/offline/skeleton_builder.h>
#include <ozz/animation/offline/animation_builder.h>
#include <ozz/animation/offline/animation_optimizer.h>
```

**核心类**:
- `RawSkeleton`: 原始骨架数据
- `RawAnimation`: 原始动画数据
- `SkeletonBuilder`: 构建运行时骨架
- `AnimationBuilder`: 构建运行动画
- `AnimationOptimizer`: 动画优化器

---

### 3. ozz_geometry (几何库)

```cpp
#include <ozz/geometry/runtime/skinning_job.h>
```

**核心类**:
- `SkinningJob`: CPU蒙皮任务

## 数据流程

```
完整的数据处理流程:

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  FBX/glTF   │────▶│    DCC      │────▶│  ozz离线工具 │
│   模型文件   │     │   工具导出   │     │  fbx2ozz    │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                                               ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   游戏渲染   │◀────│  运行时蒙皮  │◀────│  ozz运行时库 │
│  (OpenGL等)  │     │             │     │             │
└─────────────┘     └─────────────┘     └─────────────┘
                                               ▲
                                               │
                                        ┌─────────────┐
                                        │  .ozz文件   │
                                        │ (二进制数据) │
                                        └─────────────┘

步骤详解:
1. DCC工具 (Maya/Blender/Max) 导出 FBX/glTF
2. ozz离线工具链 (fbx2ozz/gltf2ozz) 导入并优化
3. 生成二进制 .ozz 文件 (skeleton, animation, track)
4. 运行时加载 .ozz 文件到内存
5. 每帧执行动画任务 (采样、混合、IK、局部转模型)
6. 获取关节模型矩阵用于蒙皮
7. GPU/CPU蒙皮后渲染
```

## 性能特点

### 内存优化

```
动画数据压缩:
┌────────────────────┬──────────────────┬──────────┐
│      数据类型       │     原始大小      │ 压缩后   │
├────────────────────┼──────────────────┼──────────┤
│ 旋转 (Quaternion)  │   16 bytes       │ 6 bytes  │
│ 平移 (Translation) │   12 bytes       │ 6 bytes  │
│ 缩放 (Scale)       │   12 bytes       │ 6 bytes  │
└────────────────────┴──────────────────┴──────────┘

压缩技术:
• 关键帧减少: 自动移除冗余关键帧
• 四元数量化: 3分量量化，第4分量恢复
• 浮点压缩: half-float 存储
• 关键帧排序: 连续存储，提高缓存命中率
• 触发帧 (iFrame): 快速定位关键帧
```

### 运行时性能

```cpp
// 典型性能数据 (Intel i7, 60个关节角色)
// 采样任务: ~0.01ms
// 混合任务: ~0.005ms
// 蒙皮任务 (1000顶点): ~0.1ms
```

## 适用场景

### 推荐使用

- **自研引擎**：需要轻量级动画系统
- **小游戏**：对包体大小敏感
- **并发量大**：需要极致性能
- **多平台**：WebAssembly、移动端
- **程序化动画**：需要IK和混合

### 不推荐使用

- **快速原型**：使用Unity/Unreal更快
- **复杂动画状态机**：需要额外实现状态机
- **物理驱动动画**：需要结合物理引擎
